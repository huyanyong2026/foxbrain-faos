## Infrastructure fixes

- System Nginx is the only public gateway for ports 80 and 443.
- Docker Nginx no longer binds host port 80 or 443; it binds only the loopback gateway target `127.0.0.1:8088` by default.
- Local VAFOX services declare both `build: context: .` and `image: foxbrain-v4:cloud` to prevent external Docker Hub pulls for the application image.
- Production `.env` is required but never generated over an existing file by deployment automation.

## Health gate

Release verification requires all of the following to pass:

```bash
docker compose ps
bash healthcheck.sh
curl -fsS http://127.0.0.1/api/health >/dev/null || curl -fsS http://127.0.0.1:3000/api/health >/dev/null
curl -fsS http://127.0.0.1/ >/dev/null || curl -fsS http://127.0.0.1:3000/ >/dev/null
ss -tlnp | awk '$4 ~ /(:80|:443)$/ {print}'
```

The `ss -tlnp` output must show no `docker-proxy` owner for public ports 80 or 443.

## Nginx status

- Host/system Nginx terminates public traffic on ports 80 and 443.
- Host Nginx proxies to the Docker Nginx loopback endpoint at `127.0.0.1:8088`.
- Docker Nginx remains internal to the host and must not expose public ports.

## Rollback method

Use the previous Git commit and rebuild services:

```bash
cd /opt/foxbrain-faos
git checkout <previous-good-commit>
docker compose up -d --build
bash healthcheck.sh
```

Alternatively, use:

```bash
APP_DIR=/opt/foxbrain-faos bash deploy.sh --rollback
```
