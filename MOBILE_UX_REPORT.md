# VAFOX Mobile UX Report

## 覆盖范围

- **gateway-web：** 将统一入口的身份切换和业务入口调整为移动端优先的信息层级；375px 起入口卡片为紧凑单列，保留清晰的跳转状态反馈。
- **ai-web：** AI 团队卡片在手机上使用可滑动、带吸附效果的卡片轨道，关键建议卡优先展示行动指令和置信度。
- **huyan-web：** CEO 驾驶舱在手机上将经营指标转换为横向可浏览的指标轨道，避免四列数据在窄屏缩小到不可读；经营建议保持单列决策流。

## 断点与验收视口

| 视口 | 预期体验 |
| --- | --- |
| 375px | 14–16px 页面边距、单列内容、可横滑的 AI / 经营指标卡片。 |
| 390px | 保持单手可读的标题、48px 左右的指标数字和完整操作文案。 |
| 414px | 卡片轨道显示下一张卡片边缘，提示可以横向浏览。 |
| 768px | 保持两列以上的桌面密度；四项数据卡片为两列布局。 |

## 企业微信与 PWA

- 页面使用 `device-width` viewport、`viewport-fit=cover` 与安全区底部间距，避免企业微信内置浏览器中的缩放及底部手势区域遮挡。
- 使用系统中文字体栈和 `-webkit-text-size-adjust: 100%`，减轻企业微信字体自动放大造成的布局漂移。
- 三个应用均提供 Web App Manifest、独立显示模式、主题色和可遮罩 SVG 图标；iOS / 企业微信收藏到桌面时也提供应用名称和状态栏配置。

## 可用性要点

- 所有按钮维持至少 36px 的可点击高度；身份切换支持水平滚动，避免窄屏文本截断。
- 横滑卡片轨道具备惯性滚动、overscroll 控制与 scroll-snap，且用 `aria-label` 告知可滑动内容。
- 桌面悬浮位移动画只在支持 hover 的环境中触发；移动端不依赖 hover 才能读到任何信息。
