# VAFOX Huyan CEO Dashboard Architecture

**文档状态：** Architecture Design / V1 Draft<br>
**定位域名：** `huyan.vafox.com`<br>
**产品定义：** CEO Private Brain + CEO Dashboard + Decision Center<br>
**设计边界：** 本文仅定义产品与逻辑架构；不执行、不授权、也不描述服务器、SSH、Docker、数据库或生产环境的操作。

---

## 0. 架构总览

Huyan 是 VAFOX CEO Control Plane 的首页驾驶舱：将企业健康、系统链路、经营事实、数据洞察、决策建议和审批事项，以一套可追溯、可解释、受权限约束的 CEO 视图呈现出来。

当前逻辑架构如下：

```text
gateway.vafox.com
        │
        ▼
huyan.vafox.com
  CEO Private Brain / CEO Dashboard / Decision Center
        │
        ▼
     Agent Hub
        │
 ┌──────┼─────────────────────────────────────────┐
 ▼      ▼              ▼           ▼              ▼
Health  Connectivity   Report      Data           CEO
Agent   Agent          Agent       Agent          Agent
```

**核心原则：** Huyan 是控制面和决策呈现层，不是生产执行面。所有首页内容均来自受治理的 Agent 报告；任何建议都必须保持为“待人类决策”的材料，不能自动改变业务、基础设施或数据。

---

## 一、CEO Dashboard 定位

### 1.1 为什么 Huyan 是 CEO 唯一入口

CEO 面对的不是单个系统的告警或单份部门报表，而是跨经营、系统、风险和组织事项的优先级判断。Huyan 应成为 CEO 的唯一入口，原因如下：

- **一个事实入口：** 以统一的时间、口径、数据新鲜度、来源证据和风险等级展示企业状态，避免 CEO 在多个系统之间自行拼接结论。
- **一个决策入口：** 将“发生了什么、为什么重要、有哪些方案、需要 CEO 决定什么”组织为闭环，而不是把原始监控和报表直接推给 CEO。
- **一个权限入口：** CEO 身份、会话、可见范围、审批动作和审计事件通过 `gateway.vafox.com` 进入 Huyan，避免直连 Agent、Core Data 或业务系统。
- **一个私有认知空间：** CEO Private Brain 保存仅供 CEO 使用的关注主题、决策上下文、偏好与已批准记忆；它增强理解，不改变企业事实源。
- **一个治理入口：** 所有 Agent 结果都要标注来源、版本、证据、新鲜度、置信度和局限；不完整、过期或冲突的信息必须明确显示，不能伪装成确定结论。

因此，Huyan 不替代企业业务系统、BI、运维平台或审批系统；它将它们的**受控、最小化、可解释输出**汇聚为 CEO 的决策工作台。

### 1.2 首页优先级

首页采用“先问题、后分析；先决策、后明细”的信息层级：

1. 先展示需要 CEO 注意的风险、异常和待决事项。
2. 再展示企业整体健康和关键经营变化。
3. 最后提供链路、指标、证据和历史趋势的下钻入口。

每个卡片都应包含状态、更新时间、数据范围、来源 Agent、证据引用和可进入的下一步；不可用、拒绝、过期和未知状态必须显式标记。

## 二、首页布局设计

### 2.1 页面结构

```text
┌──────────────────────────────────────────────────────────────────┐
│ Huyan / CEO Control Plane              数据新鲜度 │ 风险 │ 待审批 │
├──────────────────────────────────────────────────────────────────┤
│ CEO 今日简报：最重要变化、待决问题、关键建议（CEO Agent 汇总）     │
├──────────────────────┬───────────────────────────────────────────┤
│ 企业健康状态          │ CEO 决策建议 / 审批队列                   │
│ Health Agent          │ CEO Agent + Approval Flow                │
├──────────────────────┼───────────────────────────────────────────┤
│ 系统链路状态          │ 每日经营摘要                             │
│ Connectivity Agent    │ Report Agent / VAFOX_DAILY_HEALTH_REPORT │
├──────────────────────┼───────────────────────────────────────────┤
│ 企业数据分析          │ 风险中心                                 │
│ Data Agent            │ 系统风险 / 业务风险 / 数据风险           │
└──────────────────────┴───────────────────────────────────────────┘
```

布局应支持“概览 → 详情 → 证据”的三级阅读：概览页只显示最小必要信息；点击后查看某一主题的详情；证据页只向具备相应权限的角色呈现脱敏后的来源、口径和局限。

### 2.2 模块一：企业健康状态

| 项目 | 设计 |
| --- | --- |
| 数据来源 | `Health Agent` 的受控健康快照与分析报告。 |
| 首页展示 | 总体系统状态（健康 / 降级 / 异常 / 未知）、风险数量、异常提醒、最后更新时间和影响范围。 |
| 交互 | 查看风险分级、受影响的业务域、证据与人工核查建议；不提供修复或服务控制入口。 |
| 呈现规则 | `UNKNOWN`、超时、数据过期或来源缺失不可折算为“健康”。 |

### 2.3 模块二：系统链路状态

| 项目 | 设计 |
| --- | --- |
| 数据来源 | `Connectivity Agent` 的已批准链路状态报告。 |
| 首页展示 | Gateway → Huyan → Agent Hub → Agent 的组件关系图、依赖状态、断点/降级位置和报告新鲜度。 |
| 交互 | 从组件关系进入逻辑依赖详情，查看依赖声明、受影响模块与报告证据。 |
| 呈现规则 | 这是逻辑链路可观测性视图，不是网络探测、远程连接或基础设施运维控制台。 |

### 2.4 模块三：每日经营摘要

| 项目 | 设计 |
| --- | --- |
| 数据来源 | `Report Agent`。 |
| 首页展示 | `VAFOX_DAILY_HEALTH_REPORT`：今日经营脉搏、重要变化、关键异常、待关注事项、报告覆盖率和生成时间。 |
| 交互 | 阅读结构化摘要并进入来源报告；报告中的事实、推断和建议应分区呈现。 |
| 呈现规则 | 仅使用已批准来源的汇总；缺失数据、统计口径和覆盖时段必须可见。 |

### 2.5 模块四：企业数据分析

| 项目 | 设计 |
| --- | --- |
| 数据来源 | `Data Agent` 提供的最小化、聚合化数据产品。 |
| 首页展示 | 核心指标、同比/环比趋势、异常变化、机会或风险洞察，以及指标口径和数据新鲜度。 |
| 交互 | 按业务主题、时间范围和组织范围查看趋势与解释；只显示当前角色被授权的聚合粒度。 |
| 呈现规则 | 指标必须关联定义、覆盖范围和证据；不得暴露原始生产数据或作为数据库查询入口。 |

### 2.6 模块五：CEO 决策建议

| 项目 | 设计 |
| --- | --- |
| 数据来源 | `CEO Agent`，并关联 Health、Connectivity、Report、Data Agent 的证据。 |
| 首页展示 | 每条建议统一呈现：**问题、分析、方案、风险、建议**，同时显示优先级、影响、时效、置信度和待决动作。 |
| 交互 | CEO 可查看证据、要求补充分析、提交审批、驳回或委派后续人工流程。 |
| 呈现规则 | 建议与事实严格分离；CEO Agent 只提供决策辅助，不拥有自动决策、审批或执行权。 |

## 三、Agent 调用流程

### 3.1 标准流程

```text
Dashboard
    ↓ 业务问题 / 页面刷新 / 已授权查询
Agent Hub
    ↓ 身份、角色、目的、数据范围、Agent 状态与策略校验
Agent
    ↓ 受控分析与结构化输出
Report
    ↓ 来源绑定、脱敏、汇总、状态与新鲜度标记
Dashboard
```

### 3.2 调用约束

1. Dashboard 只能向 Agent Hub 发起业务语义请求，不能直连任一 Agent。
2. Agent Hub 验证调用者身份、角色、组织范围、调用目的、数据分类、Agent 版本、当前状态与审批有效期；无明确允许即拒绝。
3. Agent 只能接收其注册契约允许的最小输入，输出必须符合结构化 Report schema。
4. Report 必须携带 `report_id`、关联 ID、Agent/版本、生成时间、证据引用、数据新鲜度、置信度、不确定性、限制说明与审计引用。
5. 超时、失败、冲突或来源缺失时，Dashboard 显示 `PARTIAL`、`UNKNOWN` 或“数据过期”，并指向人工核查；不得虚构结论或自动重试非幂等请求。

### 3.3 首页编排原则

- 首页加载以缓存的已批准报告为优先，避免每次访问触发不必要的 Agent 调用。
- “刷新”仅请求只读、受策略允许的报告更新；刷新不等于触发执行。
- 跨 Agent 汇总由 Agent Hub 完成，传递最小字段和同一关联 ID；每个结论必须可回溯到来源报告。
- Agent 不可用时，其他模块仍可显示其最近有效结果，但必须标明陈旧程度和缺失影响。

## 四、权限设计

采用 **RBAC + ABAC + Capability**：角色定义可见业务域，属性策略按组织、数据分类、目的、环境、时间、风险和审批范围进一步收窄，Agent 注册能力则定义不可越过的上限。

| 主体 | 首页可见与可做 | 明确限制 |
| --- | --- | --- |
| CEO | 查看企业级已授权概览、跨域风险、决策建议与审批队列；作出批准、驳回、要求补证或委派等人类决策。 | 不因 CEO 身份而自动获得原始数据、服务器或生产执行权限。 |
| 高管 | 查看被授权组织/业务域的健康、经营摘要、趋势、风险和与自身相关的待办；可提交建议或参与指定审批。 | 不可查看 CEO Private Brain、跨域敏感事项或超出职责范围的数据。 |
| 员工 | 查看与岗位、团队和任务相关的有限 Dashboard 视图及经授权通知。 | 不可访问 CEO Dashboard 总览、决策建议明细、审批全局视图或敏感证据。 |
| Agent | 仅按已注册能力读取最小输入、输出结构化报告并写入审计事件。 | 无自主审批、委派、外发、写入、基础设施控制或生产执行权限。 |

所有权限判定均需记录主体、目的、范围、策略版本、允许/拒绝结果和关联审批。临时或敏感范围授权必须有有效期且可撤销。

## 五、数据隔离

### 5.1 三个数据域

| 数据域 | 允许内容 | 隔离规则 |
| --- | --- | --- |
| CEO Private Brain | CEO 关注主题、私人决策上下文、偏好、私有笔记与经明确允许的记忆。 | 仅 CEO 及受明确委托的最小服务可访问；不向员工 AI、普通 Agent 或通用报表回流。 |
| 员工 AI | 岗位/团队授权的任务上下文、知识与协作内容。 | 不能读取 CEO Private Brain；按员工、团队、租户和任务范围隔离。 |
| Core Data | 企业事实、治理的数据产品、指标定义、来源证据与审计记录。 | 是企业事实权威；只通过受控数据产品和最小查询契约向 Agent/界面输出，禁止任意原始数据暴露。 |

### 5.2 跨域规则

```text
Core Data ──(已批准、最小化、脱敏的数据产品)──► Agent Hub / Agent
                                                    │
CEO Private Brain ◄─(CEO 明确授权的上下文)─────────┤
                                                    ▼
                                             CEO Dashboard

员工 AI 与 CEO Private Brain：默认双向隔离
```

- CEO Private Brain 不得成为 Core Data 的替代事实源；私人观点、偏好和草稿不得反向写入企业事实。
- Employee AI 不得通过搜索、提示、报告、共享记忆或间接推断获得 CEO Private Brain 内容。
- Core Data 向 Agent 和 Dashboard 输出时遵循最小权限、字段级/聚合级脱敏、用途限制、保留期限与可审计访问。
- Dashboard 显示的每项内容都标注分类和可见范围；下载、复制、分享和外发应另受策略控制。

## 六、审批入口设计

审批入口位于 CEO 决策建议卡片和全局“待审批”队列中，用于把建议转换为可治理的人类决定，而不是自动执行器。

```text
AI 建议
  ↓ 证据、分析、方案、风险、置信度与局限
审批
  ↓ CEO / 授权审批人：批准、驳回、退回补充、委派
执行
  ↓ 仅由独立、已授权的业务工作流在审批范围内处理
结果反馈
  ↓ 结果、偏差、证据、审计记录回写为 Dashboard 的只读状态
```

### 6.1 审批对象与最小信息

每个审批项至少包含：问题陈述、建议方案与替代方案、预期影响、风险与缓解条件、证据引用、数据新鲜度、建议 Agent/版本、审批范围、有效期、责任人和审计关联 ID。

### 6.2 审批规则

- AI 可以提出建议、生成审批草稿和汇总结果，但不能批准自身建议。
- 批准只表示人类作出决策或允许进入已定义工作流，不是对任何服务器、数据或生产操作的通用授权。
- 执行状态必须由独立工作流反馈：`Pending`、`Approved`、`Rejected`、`In Progress`、`Completed`、`Failed`、`Cancelled`；Huyan 只读展示结果。
- 任何范围扩大、敏感数据使用、外部沟通或高风险事项必须重新确认审批范围，不能沿用模糊或过期批准。

## 七、风险中心

风险中心以跨模块的统一队列呈现风险，默认按严重程度、影响范围、时效性、置信度和数据新鲜度排序；同一底层事件应关联聚合，避免重复告警。

| 风险类型 | 主要来源 | 首页展示 | 建议处理 |
| --- | --- | --- | --- |
| 系统风险 | Health Agent、Connectivity Agent | 异常组件、依赖降级、影响范围、状态与证据。 | 查看逻辑依赖、要求人工核查、跟踪已批准工作流反馈。 |
| 业务风险 | Report Agent、Data Agent、CEO Agent | 经营异常、趋势偏离、潜在影响、时间窗口与方案。 | 查看数据口径与替代方案，发起人类审批或委派。 |
| 数据风险 | Data Agent、Report Agent | 数据过期、覆盖不足、口径冲突、质量异常、访问范围风险。 | 限制结论使用、要求补证或人工复核。 |

风险条目统一字段：`risk_id`、类别、标题、状态、严重度、影响、时效、证据、来源、数据新鲜度、置信度、责任人、关联审批/工作流和审计引用。风险中心不得提供绕过治理的修复、配置或生产操作按钮。

---

## 八、未来扩展

未来扩展应复用同一身份、权限、数据隔离、Agent Hub、审批和审计契约，不复制或绕过 CEO Control Plane 的治理逻辑。

| 方向 | 产品形态 | 必要前提 |
| --- | --- | --- |
| 移动端 | CEO 移动摘要、风险提醒、只读证据与受保护审批。 | 强身份验证、设备信任、最小缓存、通知脱敏和远程撤销。 |
| 小程序 | 轻量级状态查看与待审批提醒。 | 与 Web 同一授权模型；不默认暴露敏感摘要或 Private Brain。 |
| 机器人入口 | 面向 CEO 的受控问答、简报推送和审批通知。 | 明确会话身份、允许的问题范围、输出脱敏与完整审计。 |
| BI | 深度趋势、维度分析和经营复盘。 | 复用 Core Data 的治理数据产品、统一指标口径及行/列级权限。 |

任何新入口必须是 Huyan 的受治理投影，不能成为直连 Agent、Core Data、业务系统或执行系统的旁路。

---

## 九、安全边界

### 9.1 明确禁止项

Huyan CEO Control Plane、Dashboard、Agent Hub 和本文定义的 Agent 体系均**禁止**提供或触发以下能力：

- SSH、远程 shell 或服务器登录；
- 服务器修改、系统配置修改或服务控制；
- Docker / 容器创建、删除、重启、配置或镜像修改；
- 数据库查询旁路、结构修改、数据写入、删除或生产数据导出；
- 生产环境自动执行、自动修复、自动部署、自动回滚或自动变更；
- 以 AI 建议、CEO 点击、审批记录或 Dashboard 刷新为由隐式授予上述权限。

### 9.2 默认拒绝与可审计性

- 未注册 Agent、未授权数据域、未知状态、过期审批、无法验证的策略、敏感输出或不符合 schema 的请求，均默认拒绝。
- 仅呈现受控报告和审批状态；安全边界内不出现凭据、IP、主机、连接字符串、命令或可执行脚本。
- 每一次读取、汇总、建议、审批和结果反馈均应具有关联 ID、主体、时间、策略判定、数据范围、报告版本和审计引用。
- 紧急停止、权限撤销和 Agent 停用优先于缓存和可用性目标；系统应清晰显示“不可用/已撤销”，而不是展示误导性的正常状态。

---

## V1 验收结论

Huyan V1 的目标是形成 CEO 可每日使用的、只读优先且审批可追溯的 Control Plane：首页统一展示企业健康、链路状态、`VAFOX_DAILY_HEALTH_REPORT`、数据分析、决策建议和风险中心；以 Agent Hub 提供受治理报告；以明确的权限、数据隔离、审批闭环和安全边界确保“AI 辅助决策，人类负责决定，独立系统负责受控执行”。

本设计不构成任何基础设施、数据或生产执行的实施授权。
