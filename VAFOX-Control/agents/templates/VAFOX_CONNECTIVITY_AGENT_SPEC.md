# VAFOX Connectivity Agent Phase 1 标准

**文档状态：** Draft（待注册与治理审批）

**适用范围：** VAFOX-Control Agent Registry、Huyan CEO Control Plane、AI Workforce Platform 与 Core Data Brain 之间连接状态观察能力的设计。
**规范依据：** `VAFOX_AGENT_REGISTRY_STANDARD.md`、`VAFOX_AGENT_LIFECYCLE_STANDARD.md`。

## 1. Agent ID

- **Agent ID：** `agt_ops_connectivity_phase1_001`
- **名称：** `VAFOX Connectivity Agent`
- **类型：** `Connectivity`
- **版本：** `0.1.0`
- **初始状态：** `Draft`
- **部署环境：** VAFOX-Control 受控环境（仅设计阶段）
- **负责人：** 待 VAFOX Agent Registry 注册时指定；未指定前不得进入 `Registered` 状态。

Agent ID 必须全局唯一、稳定且不可复用。版本、负责人、状态、权限、数据范围或能力范围的变更必须记录于 Registry；任何扩大范围的变更必须重新审核。

## 2. 定位

VAFOX Connectivity Agent 是 VAFOX 的系统链路观察机器人。它面向未来在获批准的只读范围内观察和分析 Huyan、AI 与 Core 之间的连接状态证据，并输出可审计的连接状态结论、风险提示和人工处置建议。

Phase 1 只定义 Agent 的治理边界、输入输出契约和 Registry 模板；不连接服务器、不执行网络探测，也不接入 Huyan、AI 或 Core。它不具备任何修复、配置或执行能力。

## 3. 能力

Phase 1 仅声明以下能力：

| 能力等级 | 允许事项 | 输出 |
|---|---|---|
| Observe（观察） | 接收未来由获批准控制面提供的、脱敏或聚合的连接状态快照。 | 带 UTC 时间戳的连接状态证据摘要与引用。 |
| Analyze（分析） | 对已批准的快照进行完整性、时效性、状态一致性和链路风险分析。 | `CONNECTED`、`DEGRADED`、`DISCONNECTED` 或 `UNKNOWN` 结论。 |
| Recommend（建议） | 基于观察和分析生成仅供人工使用的核查与升级建议。 | 不可执行的人工处置建议。 |

Connectivity Agent 不具备 `Execute` 能力。Observe、Analyze 与 Recommend 不得被解释为网络访问、远程操作或系统变更权限。

### 明确禁止的动作

- SSH、SCP、SFTP、远程 shell 或远程命令执行；
- 任意远程连接、端口扫描、网络探测、DNS 查询、HTTP/API 调用或腾讯云访问；
- 网络、路由、防火墙、DNS、负载均衡或安全组配置变更；
- 对服务器、容器、服务、数据库、队列、文件、配置、凭据或生产环境的读取、写入、创建、修改、删除、启动、停止、重启、部署或回滚；
- 自动修复、自动化运维、任务分派或触发生产工作流。

## 4. 输入

每次未来的连接状态分析请求必须包含或可关联以下最小输入：

1. Agent ID、版本、调用方身份、关联 ID 与 UTC 请求时间；
2. 已批准的目标逻辑链路：`huyan_to_ai`、`huyan_to_core` 或 `ai_to_core`；
3. 由获批准控制面明确提供的脱敏/聚合连接状态快照或离线测试样本；
4. 检查配置：状态判定规则、时间窗口、数据分类和允许数据来源；
5. 有效的 Registry 状态、权限判定和审批引用。

输入不得包含密钥、令牌、凭据、完整敏感个人信息、业务原始数据、服务器地址或未经批准的数据来源。输入来源、环境、字段或审批不匹配时，必须默认拒绝处理并记录审计事件。

## 5. 输出

Connectivity Agent 必须生成结构化且可审计的连接状态报告，至少包含：

- Agent ID、版本、分析时间（UTC）、关联 ID、目标逻辑链路和检查范围；
- 连接状态结论：`CONNECTED`、`DEGRADED`、`DISCONNECTED` 或 `UNKNOWN`；
- 脱敏后的证据摘要/引用、数据新鲜度、适用的判定规则及不确定性；
- 影响等级、风险说明及仅供人工执行的下一步建议；
- Registry 状态、权限、审批和审计校验结果；
- 拒绝、错误或超时原因（如适用）。

输出不得包含凭据、密钥、令牌、服务器地址、未脱敏敏感数据或可自动执行的网络/系统变更指令。输出不得触发任何连接、检查、修复或生产动作。

## 6. 权限等级

- **权限等级：** `L0`
- **权限语义：** 无运行时系统、网络、数据、工具或执行权限；仅允许保存本 Draft 的设计性元数据和生成离线文档。
- **默认策略：** 除经正式批准并登记的范围外，一律拒绝；Phase 1 不存在任何已批准的远程或生产范围。

`L0` 不授予 SSH、远程连接、腾讯云访问、网络配置权限、服务器权限或生产环境权限。后续若申请只读观察能力，必须作为新版本提交，重新进行注册、安全审核、隔离测试和审批。

## 7. 数据范围

Phase 1 的数据范围仅限：Agent 元数据、此规范、Registry 模板和不含真实端点/凭据/生产数据的离线示例。不得访问或处理 Huyan、AI、Core、腾讯云、服务器或生产环境的实时遥测、日志、配置、网络数据、业务数据或凭据。

未来版本如需处理连接状态，必须将数据范围限定为经批准的、最小字段集的脱敏或聚合状态快照；不得默认访问原始日志、请求载荷、个人敏感信息、密钥、令牌或凭据。

## 8. 生命周期

Connectivity Agent 必须遵循 VAFOX 标准生命周期：

`Draft → Registered → Approved → Active → Suspended / Deprecated`

- **Draft：** 本规范和模板已建立；不得接入网关、连接任何目标、访问数据或接收任务。
- **Registered：** Registry 完整性校验完成；仅可进行受控注册验证与隔离测试，不得进行远程、生产或真实连接检查。
- **Approved：** 安全、测试、业务、技术与权限审批完成；可配置已批准的最小权限，但不得自动获得生产任务、网络或执行权限。
- **Active：** 仅在后续版本满足全部上线门禁并验证审计、监控和紧急停止后，才可在批准的非变更范围内运行。
- **Suspended：** 出现异常、审批失效、未授权变更或治理要求时，必须立即阻断任务、数据访问、工具调用和调度，仅保留受控审计查询。
- **Deprecated：** 撤销运行、数据、工具和调度权限后保留合规审计记录；不得恢复或复用 Agent ID。

## 9. 审批要求

`approval_required` 必须为 `true`。Phase 1 不得因文档或模板存在而获得运行授权。进入 `Registered` 前必须指定负责人；进入 `Active` 前必须完成：

1. Registry 元数据、Agent ID、版本、负责人和状态的完整性审核；
2. 安全审核，确认不存在 SSH、远程连接、腾讯云访问、网络/服务器/生产变更工具或隐式执行路径；
3. 数据与权限审核，确认数据源、字段、环境、方法和权限均遵循最小范围；
4. 隔离测试审核，验证默认拒绝、审计记录、超时处理、暂停机制以及无远程连接行为；
5. 业务、技术、安全与权限责任方的正式上线审批。

运行时必须重新校验 Agent 状态、版本、权限、数据范围和审批有效性。审批不等同于执行授权；本 Agent 在任何情况下均不拥有 Phase 1 执行授权。

## 10. 风险控制

| 风险 | 控制措施 |
|---|---|
| 设计被误用为运行时连接工具 | Phase 1 固定为 Draft 与 L0；禁止远程连接、网络探测和服务器访问。 |
| 链路观察越权读取敏感或生产数据 | 当前仅允许设计文档与无真实数据的离线示例；未来只能使用审批后的脱敏/聚合快照。 |
| 分析建议被误当作自动修复 | 输出仅限人工建议；禁止任务分派、配置修改、重启、部署、回滚和自动化运维。 |
| 审批或状态失效后仍被使用 | 默认拒绝；任何状态、身份、权限、数据范围或审批校验失败均须拒绝并审计。 |
| 审计不足导致无法追责 | `audit_enabled` 固定为 `true`；记录 Agent ID、版本、关联 ID、输入/输出摘要、校验结论、异常和 UTC 时间。 |
| 后续能力扩张绕过治理 | 网络、数据、系统或执行范围的任何扩张均须版本升级，并从 Registered 重新走审核、测试和审批。 |

## 11. Phase 1 验收边界

本版本仅交付本标准与 Registry 模板。它不实现或触发 SSH、远程连接、腾讯云访问、网络探测、网络配置修改、服务器修改、生产环境访问或任何自动修复。任何未来连接方式、数据源、阈值、集成或执行能力均必须另行完成风险评估、版本升级、注册变更、测试和审批。
