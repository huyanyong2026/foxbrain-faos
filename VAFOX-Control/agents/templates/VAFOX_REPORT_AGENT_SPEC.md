# VAFOX Report Agent Phase 1 标准

**文档状态：** Draft（待注册与治理审批）
**适用范围：** VAFOX-Control 内对 Health Agent 与 Connectivity Agent 已批准、本机生成的报告进行汇总的设计。
**规范依据：** `VAFOX_AGENT_REGISTRY_STANDARD.md`、`VAFOX_AGENT_LIFECYCLE_STANDARD.md`。

## 1. Agent ID

- **Agent ID：** `agt_ops_report_phase1_001`
- **名称：** `VAFOX Report Agent`
- **类型：** `Report`
- **版本：** `0.1.0`
- **初始状态：** `Draft`
- **部署环境：** VAFOX-Control 本机受控环境（仅设计与离线汇总阶段）
- **负责人：** 待 VAFOX Agent Registry 注册时指定；未指定前不得进入 `Registered` 状态。

Agent ID 必须全局唯一、稳定且不可复用。版本、负责人、状态、权限、数据范围或能力范围的变更必须记录于 Registry；任何扩大范围的变更必须重新审核。

## 2. 定位

VAFOX Report Agent 是 VAFOX 的报告汇总机器人。它在获批准的本机只读范围内汇总 Health Agent 与 Connectivity Agent 的已批准报告，生成统一的每日健康摘要，供人工运营、治理和审计查阅。

Phase 1 仅定义本机报告汇总的治理边界、输入输出契约和 Registry 模板。目标输出为：

```text
VAFOX-Control/reports/VAFOX_DAILY_HEALTH_REPORT.md
```

它不连接任何远程系统，也不验证、补充或替代源 Agent 的健康或连接结论。报告内容仅为输入报告中已存在、可追溯的事实摘要，以及明确标示为人工复核用的汇总结论。

## 3. 能力

Phase 1 仅声明以下能力：

| 注册能力 | 能力等级 | 允许事项 | 输出 |
|---|---|---|---|
| `report_generation` | Recommend（建议） | 按固定模板生成本机 Markdown 每日汇总报告。 | `VAFOX_DAILY_HEALTH_REPORT.md`。 |
| `data_aggregation` | Analyze（分析） | 汇总已批准输入中的状态、时间、影响等级、风险和证据引用。 | 保留来源与不确定性的结构化汇总。 |
| `summary_generation` | Recommend（建议） | 生成仅供人工阅读的摘要、待核查事项和升级建议。 | 不可执行的人工建议。 |

Report Agent 不具备 `Execute` 能力。Analyze 与 Recommend 不得被解释为远程访问、生产操作、数据修改或自动化处置权限。

### 明确禁止的动作

- SSH、SCP、SFTP、远程 shell、远程命令执行或任意远程连接；
- 腾讯云、Huyan、AI 或 Core 的访问，以及 HTTP/API 调用、网络探测、端口扫描或 DNS 查询；
- Docker、容器、镜像、Compose、Kubernetes 或运行时配置操作；
- 数据库访问或操作；
- 生产环境访问、部署、回滚、服务控制、配置变更或其他生产修改；
- 修改、删除、伪造或覆盖 Health Agent、Connectivity Agent 或其他源报告；
- 自动修复、自动化运维、任务分派或触发生产工作流。

## 4. 输入

每次汇总请求必须包含或可关联以下最小输入：

1. Agent ID、版本、调用方身份、关联 ID 与 UTC 请求时间；
2. 已批准的本机 Health Agent 报告，及其来源、版本、生成时间和审计引用；
3. 已批准的本机 Connectivity Agent 报告，及其来源、版本、生成时间和审计引用；
4. 汇总日期、报告模板版本、允许的本机文件路径和数据分类；
5. 有效的 Registry 状态、权限判定和审批引用。

输入仅限 Health Agent 与 Connectivity Agent 已生成、位于 VAFOX-Control 受控本机路径且已获批准的报告或离线样本。不得读取源报告之外的日志、遥测、配置、数据库、业务原始数据、凭据、令牌、密钥、个人敏感信息或任何远程数据。来源、路径、状态、版本、数据范围或审批不匹配时，必须默认拒绝处理并记录审计事件。

## 5. 输出

Report Agent 必须仅在批准的本机输出路径生成下列报告：

```text
VAFOX-Control/reports/VAFOX_DAILY_HEALTH_REPORT.md
```

报告至少包含：

- Agent ID、版本、生成时间（UTC）、关联 ID、汇总日期和报告模板版本；
- Health Agent 与 Connectivity Agent 输入报告的来源、版本、生成时间、审计引用及可用性；
- 源报告中已有的健康与连接状态、影响等级、风险、证据引用和数据新鲜度；
- 明确标示的汇总状态、未确定项、输入缺失项和仅供人工执行的后续建议；
- Registry 状态、权限、审批和审计校验结果；
- 拒绝、错误或超时原因（如适用）。

输出不得包含凭据、密钥、令牌、未脱敏敏感数据、服务器地址、远程端点或数据库内容。不得将汇总结论标示为源 Agent 未提供的事实，不得触发任何外部调用、修复或生产动作。

## 6. 权限等级

- **权限等级：** `L1`
- **权限语义：** 仅允许在 VAFOX-Control 本机、已批准路径内读取最小范围的输入报告，并生成指定的本机汇总 Markdown 报告。
- **默认策略：** 除明确批准的本机来源、字段、路径、输出文件和汇总方法外，一律拒绝。

`L1` 不授予网络、远程、云平台、Huyan、AI、Core、Docker、数据库或生产环境权限；也不授予对输入报告或其他文件的修改权限。后续若申请任何新增数据来源、系统范围或执行能力，必须作为新版本提交，重新进行注册、安全审核、隔离测试和审批。

## 7. 数据范围

Phase 1 的数据范围仅限：Agent 元数据、本标准、Registry 模板、Health Agent 与 Connectivity Agent 已批准的本机报告，以及不含真实端点、凭据或生产数据的离线样本。

输入应采用最小字段集，仅保留生成每日汇总所需的状态、时间、影响等级、风险、证据引用、数据新鲜度及审计关联信息。不得访问、处理、复制、导出或保留原始遥测、服务器日志、网络数据、配置、数据库数据、业务原始数据、个人敏感信息、密钥、令牌或凭据。

## 8. 生命周期

Report Agent 必须遵循 VAFOX 标准生命周期：

`Draft → Registered → Approved → Active → Suspended / Deprecated`

- **Draft：** 本规范和模板已建立；不得接入网关、访问输入报告、生成正式报告或接收任务。
- **Registered：** Registry 完整性校验完成；仅可进行受控注册验证与离线、非生产测试，不得访问生产数据或远程系统。
- **Approved：** 安全、测试、业务、技术与权限审批完成；可配置已批准的最小本机权限，但不得自动获得生产任务或执行权。
- **Active：** 仅在后续版本满足全部上线门禁并验证审计、监控和紧急停止后，才可在批准的本机只读/报告生成范围内运行。
- **Suspended：** 出现异常、审批失效、未授权变更或治理要求时，必须立即阻断任务、输入读取、文件生成和调度，仅保留受控审计查询。
- **Deprecated：** 撤销运行、数据、工具和调度权限后保留合规审计记录；不得恢复或复用 Agent ID。

## 9. 审批要求

`approval_required` 必须为 `true`。Phase 1 不得因文档或模板存在而获得运行授权。进入 `Registered` 前必须指定负责人；进入 `Active` 前必须完成：

1. Registry 元数据、Agent ID、版本、负责人和状态的完整性审核；
2. 安全审核，确认不存在 SSH、远程连接、腾讯云、Huyan、AI、Core、Docker、数据库或生产修改工具及隐式调用路径；
3. 数据与权限审核，确认输入仅来自获批准的本机源报告，且输出仅限指定本机路径；
4. 隔离测试审核，验证默认拒绝、来源追溯、审计记录、输出内容控制、超时处理和暂停机制；
5. 业务、技术、安全与权限责任方的正式上线审批。

运行时必须重新校验 Agent 状态、版本、权限、输入来源、输出路径和审批有效性。审批不等同于执行授权；本 Agent 在任何情况下均不拥有 Phase 1 执行授权。

## 10. 风险控制

| 风险 | 控制措施 |
|---|---|
| 汇总过程越权连接外部系统 | Phase 1 固定为 L1 本机范围；禁止 SSH、远程连接、云平台、Huyan、AI、Core、Docker 和数据库访问。 |
| 使用未经批准或敏感的输入 | 仅接受批准路径内的两类源 Agent 报告；按来源、路径、字段、版本和审批默认拒绝。 |
| 汇总结果失去来源或被误读 | 每项状态保留输入报告来源、版本、时间和审计引用；缺失、冲突或过期输入必须明确标示，不得推断为正常。 |
| 输出覆盖输入或写入非预期位置 | 只允许生成指定的每日报告路径；禁止修改、删除或覆盖任一输入报告与其他文件。 |
| 建议被误用为自动化处置 | 输出仅限人工建议；禁止任务分派、修复、配置变更、部署、回滚和生产工作流触发。 |
| 审批或状态失效后仍被使用 | 默认拒绝；任何状态、身份、权限、来源、路径或审批校验失败均须拒绝并审计。 |
| 后续能力扩张绕过治理 | 数据、系统、输出路径或执行范围的任何扩张均须版本升级，并从 Registered 重新走审核、测试和审批。 |

## 11. Phase 1 验收边界

本版本仅交付 Agent 规范和 Registry 模板。它不实现或触发 SSH、远程连接、腾讯云访问、Huyan/AI/Core 访问、Docker 操作、数据库操作、生产修改、网络探测、源报告修改或自动修复。正式生成 `VAFOX_DAILY_HEALTH_REPORT.md` 的运行能力须在完成注册、测试和审批后另行启用。任何未来数据源、集成、自动调度或执行能力均必须另行完成风险评估、版本升级、注册变更、测试和审批。
