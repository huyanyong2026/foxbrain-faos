# VAFOX Health Agent 第一版标准

**文档状态：** Draft（待注册与治理审批）
**适用范围：** VAFOX-Control Agent Registry、Huyan CEO Control Plane、AI Workforce Platform 与 Core Data Brain 的健康检查设计。
**规范依据：** `VAFOX_AGENT_REGISTRY_STANDARD.md`、`VAFOX_AGENT_LIFECYCLE_STANDARD.md`。

## 1. Agent 定位

VAFOX Health Agent 是 VAFOX 的首个健康检查 Agent。它仅在获授权的只读范围内收集、校验和分析健康状态证据，并输出可审计的健康结论、风险提示与人工处置建议。

Phase 1 仅定义健康检查能力，不授予任何修复或变更能力。其未来检查对象为：

- Huyan CEO Control Plane；
- AI Workforce Platform；
- Core Data Brain。

Health Agent 必须遵循默认拒绝与最小权限原则：无法验证 Agent 状态、版本、权限、审批或审计链路时，必须拒绝检查请求并记录异常。

## 2. Agent ID 与注册身份

- **Agent ID：** `agt_ops_health_phase1_001`
- **名称：** `VAFOX Health Agent`
- **类型：** `Health`
- **版本：** `0.1.0`
- **初始状态：** `Draft`
- **部署环境：** VAFOX-Control 受控环境
- **负责人：** 待 VAFOX Agent Registry 注册时指定；未指定前不得进入 `Registered` 状态。

Agent ID 必须保持全局唯一、稳定且不可复用。版本、负责人、状态、权限或能力范围发生变化时，必须在 Registry 中留下审计记录；扩大任何范围时必须重新审核。

## 3. 能力说明

Phase 1 仅声明以下能力：

| 能力等级 | 允许事项 | 输出 |
|---|---|---|
| Observe（观察） | 读取获批准的健康状态、可用性信号、运行指标、告警摘要或由控制面提供的健康快照。 | 带时间戳的原始状态摘要与证据引用。 |
| Analyze（分析） | 对已获得的只读健康证据进行完整性、阈值、连通性结果和趋势分析。 | PASS / WARN / FAIL / UNKNOWN 结论、影响范围、置信度或不确定性。 |
| Recommend（建议） | 为人工运营方生成检查后续步骤与风险升级建议。 | 不可执行的人工处置建议。 |

Health Agent 不具备 `Execute` 能力。Observe、Analyze 与 Recommend 不得被解释为系统修改权限。

### 明确禁止的动作

- SSH、SCP、SFTP 或任何远程 shell；
- 远程服务器连接、远程命令执行或端口扫描；
- Docker、容器、镜像、Compose、Kubernetes 或运行时配置的修改；
- 服务启动、停止、重启、重载或扩缩容；
- 数据库、文件、缓存、队列、业务记录、配置、密钥或任何数据的创建、修改、删除、导入、导出或迁移；
- 触发部署、回滚、自动修复或自动化运维工作流。

## 4. 权限等级与范围

- **权限等级：** `read_only`
- **数据范围：** 仅限已批准、脱敏或聚合后的健康遥测、状态码、指标、告警摘要、检查结果及审计关联信息；不读取业务原始数据、个人敏感信息、密钥、令牌或凭据。
- **执行范围：** 仅限 VAFOX-Control 本地的只读分析，以及由已批准控制面明确提供的健康快照。不得建立远程连接或调用任何可改变目标状态的接口。
- **默认策略：** 除显式批准的来源、字段、环境和方法外一律拒绝。

## 5. 输入

每次检查请求必须包含或可关联以下最小输入：

1. Agent ID、版本、调用方身份、关联 ID 与 UTC 请求时间；
2. 已批准的目标逻辑域：`huyan_ceo_control_plane`、`ai_workforce_platform` 或 `core_data_brain`；
3. 已批准的只读健康证据或控制面提供的健康快照；
4. 检查配置：指标、阈值、时间窗口、数据分类及允许的数据来源；
5. 有效的注册状态、权限判定和审批引用。

输入不得包含密钥、令牌、完整敏感个人信息或未经批准的业务原始数据。输入范围或来源不匹配时必须拒绝处理。

## 6. 输出

Health Agent 必须生成结构化且可审计的健康报告，至少包含：

- Agent ID、版本、检查时间（UTC）、关联 ID、目标逻辑域与检查范围；
- 健康结论：`PASS`、`WARN`、`FAIL` 或 `UNKNOWN`；
- 指标/检查项摘要、脱敏后的证据引用、数据新鲜度与不确定性；
- 影响等级、风险说明与仅供人工执行的下一步建议；
- 授权、审批和审计校验结果；
- 错误、拒绝或超时原因（如适用）。

输出不得包含凭据、密钥、令牌、未脱敏敏感数据，且不得直接调用修复工具或生成可自动执行的变更任务。

## 7. 生命周期

Health Agent 必须遵循 VAFOX 标准生命周期：

`Draft → Registered → Approved → Active → Suspended / Deprecated`

- **Draft：** 本规范与模板完成前的状态；不得接入网关、访问数据或接收任务。
- **Registered：** Registry 完整性校验完成；只允许受控注册验证与隔离测试，不得访问生产数据或运行生产检查。
- **Approved：** 安全、测试、业务、技术与权限审批完成；可配置最小权限，但尚未自动获得生产任务权。
- **Active：** 上线门禁、审计、监控与紧急停止机制均已验证后，才可在批准的只读范围内运行。
- **Suspended：** 发生异常、审批失效、未授权变更或治理要求时，立即阻断检查、数据访问和调度，仅保留受控审计查询。
- **Deprecated：** 撤销运行、数据、工具和调度权限后保留合规审计记录；不得恢复或复用 Agent ID。

## 8. 审批要求

`approval_required` 必须为 `true`。在进入 `Active` 前，必须完成：

1. Registry 注册完整性审核，以及 Agent ID、版本、负责人和状态核验；
2. 安全审核：确认不存在 SSH、远程连接、变更工具或隐式执行路径；
3. 数据与权限审核：确认数据来源、字段、环境和访问方法均为最小只读范围；
4. 测试审核：验证拒绝未经批准输入、审计记录、超时处理和暂停机制；
5. 业务、技术、安全与权限责任方的上线审批。

每次运行时必须重新校验 Agent 状态、版本、数据范围、权限和审批有效性。审批不等同于变更执行授权；本 Agent 在任何情况下均不拥有执行授权。

## 9. 风险与控制

| 风险 | 控制措施 |
|---|---|
| 健康结论基于过期或不完整证据 | 输出数据新鲜度与 `UNKNOWN` 状态；证据无法验证时拒绝给出通过结论。 |
| 健康检查越权访问敏感数据 | 仅接受批准的脱敏/聚合健康数据；按字段、来源、环境和方法执行默认拒绝。 |
| 诊断能力被扩展为远程运维 | 固定为无 SSH、无远程连接、无变更工具的本地只读范围；能力扩大必须重新注册和审批。 |
| 误把建议当作自动修复 | 输出仅限人工建议，禁止触发部署、重启、Docker 修改或数据修改。 |
| 审计不足导致无法追责 | 对每次调用记录身份、版本、关联 ID、授权判定、输入摘要、输出摘要、异常与 UTC 时间。 |
| Agent 异常或审批失效后仍运行 | 运行时校验失败即拒绝并告警；通过 Registry 将 Agent 切换为 `Suspended`。 |

## 10. Phase 1 验收边界

本版本仅交付 Agent 规范和 Registry 模板。它不实现网络探测、远程访问、服务控制、容器操作、数据写入或自动修复。任何后续连接方式、数据源、阈值、监控集成或执行能力都必须另行完成风险评估、版本升级、注册变更、测试和审批。
