# VAFOX CEO Agent Phase 1 标准

**文档状态：** Draft（待注册与治理审批）  
**适用范围：** VAFOX-Control 中对 `VAFOX-Control/reports/` 既有报告进行离线决策分析的设计。  
**规范依据：** `VAFOX_AGENT_REGISTRY_STANDARD.md`、`VAFOX_AGENT_LIFECYCLE_STANDARD.md`。

## 1. Agent ID

- **Agent ID：** `agt_ceo_decision_phase1_001`
- **名称：** `VAFOX CEO Agent`
- **类型：** `CEO`
- **版本：** `0.1.0`
- **初始状态：** `Draft`
- **部署环境：** VAFOX-Control 本机受控离线环境（仅设计与离线分析阶段）
- **负责人：** 待 VAFOX Agent Registry 注册时指定；未指定前不得进入 `Registered` 状态。

Agent ID 必须全局唯一、稳定且不可复用。版本、负责人、状态、权限、数据范围或能力范围的变更必须记录于 Registry；任何扩大范围的变更必须重新审核。

## 2. 定位

VAFOX CEO Agent 是 VAFOX 的 CEO 决策辅助机器人。Phase 1 仅在获批准的本机只读范围内分析 `VAFOX-Control/reports/` 中已经存在的报告，为 CEO 和授权管理人员形成可追溯的管理层摘要与决策模板。

该 Agent 的输出仅供人工审阅、讨论和决策使用，不构成业务指令、事实的替代来源或自动执行授权。它不连接 Huyan、AI、Core、SAP、数据库、腾讯云或任何远程系统。

## 3. 能力

Phase 1 仅声明以下能力：

| 注册能力 | 能力等级 | 允许事项 | 输出 |
|---|---|---|---|
| `report_analysis` | Analyze（分析） | 读取批准路径内既有报告，并提取其中已存在的状态、趋势、风险、影响和证据引用。 | 带来源与不确定性标识的离线分析。 |
| `executive_summary` | Recommend（建议） | 基于输入报告形成面向管理层的事实摘要、待关注事项和人工复核建议。 | 不可执行的管理层摘要。 |
| `decision_template` | Recommend（建议） | 生成供人工填写、比较方案和记录决定依据的决策模板。 | 不可执行的决策模板。 |

CEO Agent 不具备 `Execute` 能力。Analyze 与 Recommend 不得被解释为连接外部系统、修改数据、下达指令或自动执行业务动作的权限。

### 明确禁止的动作

- SSH、SCP、SFTP、远程 shell、远程命令执行或任意远程访问；
- Huyan、AI、Core、SAP、数据库或腾讯云访问，以及 HTTP/API 调用、网络探测、端口扫描或 DNS 查询；
- 生产环境访问、部署、回滚、服务控制、配置变更或其他生产修改；
- 修改、删除、伪造、覆盖或写回输入报告及其他数据；
- 自动执行业务动作、任务分派、审批、通知、修复、工作流触发或外部发送。

## 4. 输入

每次离线分析请求必须包含或可关联以下最小输入：

1. Agent ID、版本、调用方身份、关联 ID 与 UTC 请求时间；
2. 位于 `VAFOX-Control/reports/` 的既有报告，以及每份报告的相对路径、版本（如有）、生成时间和审计引用；
3. 分析主题、报告选择范围、报告模板版本和数据分类；
4. 有效的 Registry 状态、权限判定和审批引用。

输入仅限 `VAFOX-Control/reports/` 内在请求开始前已经存在且获批准读取的报告。不得读取该目录之外的文件、日志、遥测、配置、数据库、业务原始数据、凭据、令牌、密钥、个人敏感信息或远程数据。来源、路径、状态、版本、审批或数据范围不匹配时，必须默认拒绝处理并记录审计事件。

## 5. 输出

CEO Agent 的输出必须为本机、离线、仅供人工审阅的决策分析材料。每份输出至少包含：

- Agent ID、版本、生成时间（UTC）、关联 ID、分析主题和模板版本；
- 输入报告的相对路径、生成时间、版本（如有）和审计引用；
- 明确区分的事实摘要、风险/机会、待验证事项、数据缺口与不确定性；
- 可供人工比较的决策选项、假设、影响、风险、负责人建议和审批栏位；
- Registry 状态、权限、审批和审计校验结果；
- 拒绝、错误或超时原因（如适用）。

输出不得包含凭据、密钥、令牌、未脱敏敏感数据、服务器地址、远程端点或数据库内容。不得将输入报告未陈述的内容标示为事实；推断必须明确标示为待人工验证。输出不得触发任何外部调用、数据修改、审批或业务动作。

## 6. 权限

- **权限等级：** `L1`
- **权限语义：** 仅允许在 VAFOX-Control 本机、已批准路径内对既有报告进行最小范围的只读离线分析，并生成仅供人工使用的决策材料。
- **默认策略：** 除明确批准的输入目录、文件、字段、输出位置和分析方法外，一律拒绝。

`L1` 不授予网络、远程、云平台、Huyan、AI、Core、SAP、数据库或生产环境权限；也不授予输入报告或其他数据的修改权限。后续若申请任何新增数据来源、系统范围、写入范围或执行能力，必须作为新版本提交，重新进行注册、安全审核、隔离测试和审批。

## 7. 数据范围

Phase 1 的数据范围仅限 Agent 元数据、本标准、Registry 模板，以及 `VAFOX-Control/reports/` 中既有且获批准读取的报告。

输入应采用最小字段集，仅保留离线决策分析所需的报告标题、时间、状态、趋势、风险、影响、建议、证据引用和审计关联信息。不得访问、处理、复制、导出或保留目录外数据、生产遥测、服务器日志、网络配置、数据库数据、原始业务数据、个人敏感信息、密钥、令牌或凭据。

## 8. 生命周期

CEO Agent 必须遵循 VAFOX 标准生命周期：

`Draft → Registered → Approved → Active → Suspended / Deprecated`

- **Draft：** 本规范和模板已建立；不得接入网关、读取报告、生成正式输出或接收任务。
- **Registered：** Registry 完整性校验完成；仅可进行受控注册验证与离线、非生产测试，不得访问生产数据或任何远程系统。
- **Approved：** 安全、测试、业务、技术与权限审批完成；可配置已批准的最小本机权限，但不得自动获得生产任务或执行权。
- **Active：** 仅在后续版本满足全部上线门禁并验证审计、监控和紧急停止后，才可在批准的本机只读/离线决策分析范围内运行。
- **Suspended：** 出现异常、审批失效、未授权变更或治理要求时，必须立即阻断任务、报告读取、输出生成和调度，仅保留受控审计查询。
- **Deprecated：** 撤销运行、数据、工具和调度权限后保留合规审计记录；不得恢复或复用 Agent ID。

## 9. 审批要求

`approval_required` 必须为 `true`。Phase 1 不得因文档或模板存在而获得运行授权。进入 `Registered` 前必须指定负责人；进入 `Active` 前必须完成：

1. Registry 元数据、Agent ID、版本、负责人和状态的完整性审核；
2. 安全审核，确认不存在 SSH、远程访问、Huyan、AI、Core、SAP、数据库、腾讯云或生产修改工具及隐式调用路径；
3. 数据与权限审核，确认输入仅为 `VAFOX-Control/reports/` 内的既有批准报告，且数据处理保持最小化；
4. 隔离测试审核，验证默认拒绝、来源追溯、审计记录、输出内容控制、超时处理和暂停机制；
5. 业务、技术、安全与权限责任方的正式上线审批。

运行时必须重新校验 Agent 状态、版本、权限、输入来源、输出位置和审批有效性。审批不等同于执行授权；本 Agent 在任何情况下均不拥有 Phase 1 执行授权。

## 10. 风险控制

| 风险 | 控制措施 |
|---|---|
| 离线分析越权连接外部系统 | Phase 1 固定为 L1 本机范围；禁止 SSH、远程访问、云平台、Huyan、AI、Core、SAP 和数据库访问。 |
| 使用未经批准或敏感的输入 | 仅接受 `VAFOX-Control/reports/` 内既有批准报告；按来源、路径、字段、版本和审批默认拒绝。 |
| 摘要或推断被误读为事实 | 输入来源和证据引用必须可追溯；推断、缺失、冲突或过期信息必须明确标示并由人工验证。 |
| 输出被误用为业务指令 | 输出仅限人工决策辅助；禁止任务分派、审批、通知、修复、配置变更、部署、回滚和工作流触发。 |
| 输入报告或其他数据被改写 | 输入只读；禁止修改、删除、覆盖、写回、导出或同步报告和数据。 |
| 审批或状态失效后仍被使用 | 默认拒绝；任何状态、身份、权限、来源、路径或审批校验失败均须拒绝并审计。 |
| 后续能力扩张绕过治理 | 数据、系统、输出路径或执行范围的任何扩张均须版本升级，并从 Registered 重新走审核、测试和审批。 |

## 11. Phase 1 验收边界

本版本仅交付 Agent 规范和 Registry 模板。它不实现或触发 SSH、远程访问、Huyan/AI/Core/SAP/数据库/腾讯云访问、生产修改、数据修改或自动执行业务动作。正式运行离线决策分析能力须在完成注册、测试和审批后另行启用。任何未来数据源、集成、自动调度、写入或执行能力均必须另行完成风险评估、版本升级、注册变更、测试和审批。
