# VAFOX Data Agent Phase 1 标准

**文档状态：** Draft（待注册与治理审批）
**适用范围：** VAFOX-Control Agent Registry 中 VAFOX Data Agent 的 Phase 1 离线设计。
**规范依据：** `VAFOX_AGENT_REGISTRY_STANDARD.md`、`VAFOX_AGENT_LIFECYCLE_STANDARD.md`。

## 1. Agent ID

- **Agent ID：** `agt_ops_data_phase1_001`
- **名称：** `VAFOX Data Agent`
- **类型：** `Data`（受治理批准的扩展类型）
- **版本：** `0.1.0`
- **初始状态：** `Draft`
- **部署环境：** VAFOX-Control 本机受控设计环境
- **负责人：** 待 VAFOX Agent Registry 注册时指定；在指定前不得进入 `Registered` 状态。

Agent ID 必须全局唯一、稳定且不可复用。版本、负责人、状态、能力、权限、数据范围或执行范围的变更必须在 Registry 留存审计；扩大任何范围时必须重新审核。

## 2. 定位

VAFOX Data Agent 是 VAFOX 的企业数据分析机器人。它的职责是基于**已批准的离线、非生产、脱敏或合成样本**定义数据查询、指标分析和洞察生成的标准输入输出契约，为人工决策提供可追溯的分析草案。

Phase 1 只交付设计规范与 Registry 模板：不接入 Core，不访问数据库，不连接 SAP、腾讯云或任何远程系统，也不处理真实生产数据。所有洞察均为人工复核用建议，不构成业务事实、自动决策或执行指令。

## 3. 能力

Phase 1 仅声明以下能力：

| 注册能力 | 能力等级 | 允许事项 | 输出 |
|---|---|---|---|
| `data_query` | Observe | 在批准的离线样本中按预定义字段筛选、检索和聚合。 | 可追溯的查询摘要。 |
| `metric_analysis` | Analyze | 对批准的聚合样本计算、比较和解释业务指标。 | 指标结果、口径、时间范围与局限性。 |
| `insight_generation` | Recommend | 基于查询与指标结果生成带不确定性说明的人工洞察。 | 非执行性的分析洞察与待核查项。 |

Data Agent 不具备 `Execute` 能力。Observe、Analyze 与 Recommend 不得被解释为数据源连接、数据修改、系统调用、生产访问或自动化处置权限。

### 明确禁止的动作

- SSH、SCP、SFTP、远程 shell、远程命令执行、远程连接、网络探测或端口扫描；
- 腾讯云、Core、SAP、数据库、数据仓库、消息队列、文件服务或任何远程 API 的访问；
- 读取、写入、创建、更新、删除、导入、导出、迁移或复制生产数据、业务记录、配置、凭据、密钥或令牌；
- 生产环境访问、部署、回滚、服务控制、配置修改、工作流触发或其他生产修改；
- 自动执行数据查询、告警、决策、任务分派或修复动作。

## 4. 输入

每次离线分析请求必须包含或可关联以下最小输入：

1. Agent ID、版本、调用方身份、关联 ID 与 UTC 请求时间；
2. 已批准的离线数据样本或合成数据集，以及其来源、版本、数据分类和脱敏声明；
3. 分析请求：允许的字段、指标口径、时间窗口、过滤条件与预期输出格式；
4. 有效的 Registry 状态、权限判定和审批引用。

输入仅限 Agent 元数据、设计文档，以及不含真实端点、凭据、个人敏感信息或生产数据的获批准离线样本。输入来源、字段、版本、数据分类或审批不匹配时，必须默认拒绝处理并写入审计记录。

## 5. 输出

Data Agent 的输出必须是结构化、可审计且仅供人工复核的离线分析结果，至少包含：

- Agent ID、版本、生成时间（UTC）、关联 ID、样本版本与分析范围；
- 查询或筛选条件摘要、允许字段、指标口径、时间窗口与数据新鲜度声明；
- 聚合结果、分析结论、证据引用、置信度或不确定性、局限性与待核查项；
- 非执行性的人工建议，以及 Registry、权限、审批与审计校验结果；
- 拒绝、错误或超时原因（如适用）。

输出不得包含原始生产数据、数据库内容、凭据、密钥、令牌、远程端点、个人敏感信息或可直接执行的生产操作。不得将样本分析结论表述为已验证的生产事实，不得触发任何外部调用或变更。

## 6. 权限等级

- **权限等级：** `L0`
- **权限语义：** 仅允许本机设计文档编制与对获批准离线非生产样本的只读分析；不授予运行时数据访问权限。
- **默认策略：** 除显式批准的本机资料、样本、字段、分析方法和输出格式外，一律拒绝。

`L0` 不授予网络、远程、腾讯云、Core、SAP、数据库、数据仓库、生产环境、写入、导出或执行权限。任何未来数据源、系统连接、原始数据处理或执行能力申请，必须以新版本重新注册、隔离测试和审批。

## 7. 数据范围

Phase 1 数据范围仅限：Agent 元数据、本标准、Registry 模板，以及经批准的离线、非生产、合成、脱敏或聚合数据样本。样本必须不含真实生产记录、数据库转储、业务原始数据、个人敏感信息、凭据、密钥、令牌、服务器地址或远程端点。

数据处理必须采用最小字段集，仅保留完成指定分析所必需的字段、指标、时间范围与审计关联信息。不得访问、保留、复制、导出或跨域传输任何未批准数据。

## 8. 生命周期

Data Agent 必须遵循 VAFOX 标准生命周期：

`Draft → Registered → Approved → Active → Suspended / Deprecated`

- **Draft：** 已完成规范与模板；不得接入网关、连接数据源、访问数据或接收任务。
- **Registered：** Registry 完整性校验完成；仅可进行受控注册验证与离线非生产测试，不得访问生产数据或远程系统。
- **Approved：** 安全、测试、业务、技术与权限审批完成；可配置获批准的最小权限，但不得自动获得生产任务或数据连接权。
- **Active：** 仅在后续版本完成上线门禁、审计、监控与紧急停止验证后，才可在批准范围内运行；Phase 1 不进入具备外部连接的 Active 模式。
- **Suspended：** 出现异常、审批失效、未授权变更或治理要求时，必须立即阻断任务、样本读取和调度，仅保留受控审计查询。
- **Deprecated：** 撤销运行、数据、工具和调度权限后保留合规审计记录；不得恢复或复用 Agent ID。

## 9. 审批要求

`approval_required` 必须为 `true`。Phase 1 的规范或模板不构成运行授权。进入 `Registered` 前必须指定负责人；任何未来进入 `Active` 的版本前，必须完成：

1. Registry 元数据、Agent ID、版本、负责人和状态的完整性审核；
2. 安全审核，确认不存在 SSH、远程连接、腾讯云、Core、SAP、数据库或生产修改工具及隐式调用路径；
3. 数据与权限审核，确认数据来源、字段、样本分类、环境和访问方法均为最小离线只读范围；
4. 隔离测试审核，验证默认拒绝、数据脱敏、审计记录、超时处理和暂停机制；
5. 业务、技术、安全与权限责任方的正式上线审批。

每次运行时必须重新校验 Agent 状态、版本、数据范围、权限和审批有效性。审批不等同于数据连接或执行授权。

## 10. 风险控制

| 风险 | 控制措施 |
|---|---|
| 离线分析被扩展为访问企业真实数据 | 固定 L0 与无网络边界；仅允许批准的合成、脱敏或聚合非生产样本，新增来源必须重新审批。 |
| 数据查询越权访问 Core、SAP 或数据库 | 明确禁止 Core、SAP、数据库及任何远程 API/连接；来源、字段和审批不匹配时默认拒绝。 |
| 样本或输出泄露敏感信息 | 最小字段集、数据分类和脱敏声明为输入门禁；输出禁止原始数据、凭据、令牌、端点与个人敏感信息。 |
| 洞察被误读为生产事实或自动决策 | 输出必须标示样本范围、证据、置信度、不确定性和待核查项；只提供人工建议，禁止自动触发动作。 |
| 审计不足导致无法追责 | 每次调用记录身份、版本、关联 ID、输入/输出摘要、授权与审批判定、拒绝原因和 UTC 时间。 |
| 审批或状态失效后仍被使用 | 运行时校验失败即拒绝并审计；异常时通过 Registry 切换为 `Suspended`。 |

## 11. Phase 1 验收边界

本版本仅交付 Agent 规范和 Registry 模板。它不实现或触发 SSH、远程连接、腾讯云访问、Core 访问、SAP 访问、数据库访问、真实数据访问、生产修改、网络探测、数据导出或自动化执行。任何后续连接、真实数据范围、生产部署、调度或执行能力，都必须另行完成风险评估、版本升级、注册变更、隔离测试和审批。
