# VAFOX Enterprise OS Master Plan

Version: Alpha 0.1
Target Domain: huyan.vafox.com
Owner: Huohu Fox / 火狐狸

---

## 1. Mission

VAFOX exists to become the enterprise second brain of Huohu Fox.

It must preserve enterprise knowledge, connect enterprise data, understand enterprise relationships, remember decisions, and assist the CEO in making better decisions.

## 2. Product Boundary

### In Scope Now

- huyan.vafox.com
- CEO Brain
- VAFOX Drive
- Object Engine
- Knowledge Pipeline
- Global Search
- Timeline
- CEO Dashboard
- SAP data import / sync foundation

### Out of Scope Now

- ai.vafox.com employee platform
- public SaaS platform
- external customer portal
- supplier portal
- mobile app

These will be planned later after CEO Brain is stable.

## 3. Core Philosophy

### 3.1 One Enterprise, One Brain

All enterprise data, files, knowledge, memories and decisions must enter one unified brain.

### 3.2 Everything is an Object

Stores, employees, brands, products, suppliers, customers, contracts, files, videos, meetings and decisions are all objects.

### 3.3 Every File Has Value

There are no useless files. There are only files not yet understood by AI.

### 3.4 AI First

Every module must be designed with AI understanding, classification, search, summarization and reasoning capability from the beginning.

### 3.5 Data Belongs to the Enterprise

All uploaded files, extracted text, AI summaries, object relations, vectors, graph data and decision records must be exportable, backupable and migratable.

## 4. 20 Core Engines

1. Drive Engine: upload, store, preview, classify and process files.
2. Object Engine: unified enterprise object model.
3. Knowledge Engine: convert files and data into knowledge.
4. Memory Engine: preserve enterprise decisions, history and reasons.
5. Timeline Engine: build time-based enterprise history.
6. Search Engine: global search across all objects and files.
7. Graph Engine: connect objects into knowledge graph.
8. Agent Engine: manage AI agents.
9. Decision Engine: provide structured decision support.
10. Dashboard Engine: CEO daily view.
11. SAP Engine: SAP data import and sync.
12. Analytics Engine: sales, inventory, finance and brand analysis.
13. OCR Engine: image and scanned document understanding.
14. Media Engine: photos, videos and audio handling.
15. Workflow Engine: tasks and automation workflows.
16. Content Engine: public content generation.
17. Permission Engine: role and data security.
18. Plugin Engine: future integrations.
19. API Engine: external and internal API layer.
20. Automation Engine: scheduled jobs and background processing.

## 6. Definition of Done

Every sprint is complete only when:

- product requirements are implemented
- UI is usable
- backend API works
- database migration is stable
- errors are handled
- existing functions are not broken
- basic tests are added or manual test checklist is completed
- deployment succeeds
- rollback path exists
- README or sprint document is updated

## 7. 90-Day Target
