## 一、VAFOX 2.0 最终定位

VAFOX 2.0 的最终形态是 **VAFOX Outdoor Intelligence Ecosystem（VAFOX 户外智能生态系统）**。

VAFOX 从一家户外零售企业，升级为连接 **人、商品、数据、知识、AI、组织与客户** 的企业智能生态系统。其核心不是单一应用、单一模型或单一自动化流程，而是以 **VAFOX OS** 为统一底座，将经营事实、组织能力、客户体验和 AI 劳动力组织为可持续演进的智能网络。

```text
户外产业 / 商品 / 场景
          +
人 / 组织 / 客户
          +
数据 / 知识 / AI
          ↓
VAFOX Outdoor Intelligence Ecosystem
          ↓
VAFOX Enterprise AI OS
```

VAFOX Enterprise AI OS 的长期使命是：让企业能够持续完成“感知—理解—决策—协同—执行—复盘—学习”的经营闭环，并把每一次有效经营与服务经验沉淀为组织能力。

## 五、数据架构

```text
SAP B1
  ↓
Core Data Brain
  ↓
Data Pipeline
  ↓
Data Agent
  ↓
CEO AI / Employee AI / Customer AI
```

### 数据层模型

| 层级 | 定义 | 关键控制 |
|---|---|---|
| 源系统层 | SAP B1 是关键经营业务记录的权威来源；其他业务来源按数据契约接入。 | 源系统事实不被 AI 输出覆盖或篡改。 |
| Core Data Brain | 企业事实、主数据、指标和数据产品的统一可信中枢。 | 统一口径、质量、血缘、分类、版本与访问策略。 |
| Data Pipeline | 将原始数据转化为标准化、可分析、可服务的数据产品。 | 处理规则可追溯、可测试、可回溯。 |
| Data Agent | 受控查询、质量解释、分析与派生数据准备。 | 仅按目的、时间、行列与字段范围访问最小数据。 |
| AI 消费层 | CEO AI、Employee AI、Customer AI 使用受权的数据产品形成洞察与服务。 | AI 结果标识为派生智能，并携带数据来源、时间与限制。 |

**数据原则：** 事实与推理分离；数据访问与业务写入分离；数据产品与原始存储分离；任何高风险行动均通过独立审批与业务工作流完成。

## 六、知识架构

```text
合同 / 制度 / 品牌 / 产品 / 经验 / 案例 / 路线 / 攻略 / 决策
                              ↓
                       Knowledge Brain
                              ↓
                           AI 服务
```

Knowledge Brain 是 VAFOX 的长期企业记忆，也是面向户外专业服务的可信知识底座。知识对象必须保留来源、所有者、版本、密级、有效期、适用范围与引用关系。

| 知识类型 | 核心价值 | 主要 AI 服务 |
|---|---|---|
| 合同、制度、决策 | 合规、治理与经营依据。 | 受权检索、条款理解、决策依据与风险提示。 |
| 品牌、产品、案例 | 品牌一致性、商品专业度与经营经验。 | 员工问答、商品推荐、内容与销售辅助。 |
| 经验、路线、攻略 | 户外场景能力与客户服务差异化。 | Outdoor AI Guide、行前建议、场景化服务。 |

**知识服务原则：** 先校验权限和时效，再进行检索与生成；输出必须能够追溯来源；过期、冲突或无来源内容不得被包装为确定结论。

## 七、人才架构

```text
员工
  ↓
People Brain
  ↓
AI 成长助手
  ↓
组织能力提升
```

People Brain 将个人成长与组织能力连接为持续循环：员工在岗位中学习与反馈，AI 成长助手按角色和目标提供受控支持，组织据此识别能力缺口并持续完善岗位知识与培养体系。

| 层次 | 架构能力 | 结果 |
|---|---|---|
| 员工 | 角色、技能、目标、经验与学习反馈。 | 可被尊重、可持续积累的个人能力上下文。 |
| People Brain | 能力模型、岗位知识包、成长路径与组织能力地图。 | 对人和岗位的结构化、最小必要理解。 |
| AI 成长助手 | 个性化学习、任务辅导、知识推荐、反馈与复盘支持。 | 更快形成岗位胜任力与协同能力。 |
| 组织能力提升 | 汇总匿名化或授权范围内的能力信号。 | 优化培养、知识建设、人才配置与组织学习。 |

人才数据采用严格分级、目的限定与最小可见原则；AI 成长助手不替代人事判断，不自动执行敏感人事决策。

## 九、Control Plane 治理

Control Plane 是 VAFOX Enterprise AI OS 的统一治理平面，负责确保 AI Workforce 的每一个能力都可识别、可控制、可观测、可审计和可停止。

| 治理能力 | 定义 |
|---|---|
| Agent Registry | 为每个 Agent 登记唯一 ID、职责、所有者、版本、输入输出、工具、数据范围、风险等级与生命周期。 |
| Runtime | 提供受控运行环境、任务编排、队列、超时、隔离、失败处理与紧急停止。 |
| 权限 | 将用户委派、Agent 能力、数据分类、工具范围、环境与任务目的共同纳入策略决策。 |
| 日志 | 记录请求、上下文摘要、模型与工具调用、状态、输出、错误与关联标识。 |
| 审计 | 建立身份、数据访问、审批、执行、版本和安全事件的全链路可追溯记录。 |
| 版本 | 管理 Agent、提示词、工作流、工具、模型和策略的版本、发布状态与回退关系。 |
| 成本 | 观测模型、工具、数据处理和运行资源消耗，并按 Agent、任务、域和业务价值归因。 |
| 安全 | 执行输入输出防护、Secret 隔离、策略拦截、异常检测、审批与安全告警。 |

## 十、安全体系

VAFOX 的安全体系以“身份可信、授权最小、数据隔离、AI 可控、全程可审计”为底线。

| 安全能力 | 架构要求 |
|---|---|
| SSO | 所有人类用户、应用与受管服务通过统一身份体系进入受保护域。 |
| MFA | CEO、高管和高风险操作采用强化 MFA；风险变化触发追加验证。 |
| RBAC | 按 CEO、管理者、员工、客户、Agent 与运营角色确定能力上限。 |
| ABAC | 依据组织、数据分类、资源、目的、时间、环境、设备与风险属性收窄实际访问范围。 |
| 数据隔离 | CEO 私有域、员工工作域、客户域、核心数据域和 Agent 运行域使用独立策略与最小化跨域传递。 |
| AI 安全 | Agent 使用短期身份与受限工具；输入输出受保护；模型结果不能自动获得业务写入权。 |
| 审计 | 记录认证、授权、数据访问、Agent 行为、审批、策略与版本事件，并支持追溯与独立审查。 |

**安全决策表达式：**

```text
允许访问 = 有效身份 ∩ 强认证 ∩ 角色权限 ∩ 属性策略 ∩ 资源策略 ∩ 已批准目的 ∩ Agent 能力（非人类主体）
```

任何条件缺失、无效或不可验证时，系统默认拒绝。

## 十一、移动生态

VAFOX OS 采用 **手机优先（Mobile First）** 的体验策略：核心任务、提醒、审批、服务与行动入口首先适配移动场景；复杂配置、治理、深度分析和批量管理由桌面端补充。

| 移动入口 | 核心体验 |
|---|---|
| CEO | 经营摘要、风险提醒、决策待办、授权审批、行动追踪与 CEO 私有问答。 |
| 员工 | AI 工作助手、岗位任务、知识问答、学习成长、协作消息与反馈。 |
| 客户 | VAFOX ID、商品发现、会员服务、订单支持、户外 AI Guide 与个性化旅程。 |

移动体验必须继承统一身份、MFA、最小权限、隐私同意与审计能力，不以便利性降低安全与治理标准。

## 十四、最终愿景

**VAFOX Enterprise AI OS** 是 VAFOX 面向未来十年的长期战略基础设施。

它成为连接户外产业、人、商品、数据、知识和 AI 的统一智能底座：

```text
让企业更懂经营
让组织持续学习
让员工获得能力放大
让客户获得可信赖的户外服务
让 AI 在安全、治理与责任边界内创造长期价值
```

最终，VAFOX 不仅运营户外零售业务，更运营一个能够持续学习、持续协同、持续服务产业与客户的 **Outdoor Intelligence Ecosystem**。
