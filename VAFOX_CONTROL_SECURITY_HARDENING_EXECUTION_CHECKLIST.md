# VAFOX Control Plane 安全加固执行前检查清单

> **用途：** 本清单是 `control.vafox.com` 安全加固的执行前门禁、逐步验证和验收记录标准。仅用于审批后的受控变更准备与记录，**不构成执行授权，不包含也不触发任何服务器、腾讯云或网络操作**。
>
> **当前状态：Approved with Conditions（有条件批准）。** 仅当本清单中的全部条件、审批、备份和恢复路径均被确认并记录后，授权执行负责人才能在批准窗口内开始变更。任一必填项失败、未知或不一致，必须停止，不得继续。
>
> **安全记录规则：** 使用 UTC；只记录受控系统引用、脱敏证据、指纹与校验和。严禁记录密码、私钥、Token、API Key、恢复密钥或其他明文 Secret。

## 四、执行顺序

> **通用规则：** 每个 Step 必须先记录执行前状态和备份位置，再进行获批变更，随后立刻完成验证。验证失败时，不得进入下一步；按第 5 节对应回滚点处理。此处仅定义检查项，不提供或执行服务器命令。

### Step 1：创建管理员账号

| 项目 | 检查内容 |
| --- | --- |
| 变更前提 | 实名管理员身份、最小 sudo 范围、管理组和授权公钥引用均已批准；SSH 及 sudo 备份通过。 |
| 执行检查 | [ ] 已按批准方案创建非 root、可归属个人的管理员账号。<br>[ ] 未使用共享账号或 `NOPASSWD: ALL` 等权限捷径。<br>[ ] 保持现有管理会话可用。 |
| 验证 | [ ] **管理员 SSH 登录成功**：从批准来源使用新管理员账号建立独立会话。<br>[ ] 管理员身份和权限与审批一致。 |
| 放行条件 | 新管理员独立登录成功且最小权限验证通过。 |

### Step 2：配置 SSH Key

| 项目 | 检查内容 |
| --- | --- |
| 变更前提 | 管理员账号验证成功；批准的公钥指纹/密钥管理引用已核对；私钥未写入主机、文档或报告。 |
| 执行检查 | [ ] 已为获批管理员配置独立 SSH Key 认证。<br>[ ] 授权密钥路径、属主和权限符合安全要求。<br>[ ] 未关闭现有已验证会话。 |
| 验证 | [ ] **Key 登录成功**：从批准来源以登记 Key 新建独立 SSH 会话。<br>[ ] 成功证据仅记录指纹、时间和受控日志引用。 |
| 放行条件 | Key 登录成功，可在第二会话中保持稳定管理访问。 |

### Step 3：关闭 root 远程登录

| 项目 | 检查内容 |
| --- | --- |
| 变更前提 | 管理员 Key 登录已在独立会话验证成功；SSH 配置备份和 SSH 失败恢复路径均已确认。 |
| 执行检查 | [ ] 仅按批准配置禁止 root 远程登录。<br>[ ] 变更前后均进行 SSH 配置有效性检查。<br>[ ] 不断开已验证管理员会话。 |
| 验证 | [ ] **管理员仍可登录**：从批准来源使用管理员 Key 再次建立独立会话。<br>[ ] root 远程登录拒绝结果已记录（不得记录凭据）。 |
| 放行条件 | 管理员登录持续可用，root 远程登录已按批准策略拒绝。 |

### Step 4：关闭密码登录

| 项目 | 检查内容 |
| --- | --- |
| 变更前提 | 管理员 Key 登录及 root 登录禁用验证均通过；保留已验证会话和带外恢复路径。 |
| 执行检查 | [ ] 仅按批准配置关闭密码及交互式密码认证。<br>[ ] SSH 配置有效性检查已完成。 |
| 验证 | [ ] **SSH Key 正常**：管理员使用登记 Key 重新建立独立会话。<br>[ ] 密码登录按批准策略拒绝。 |
| 放行条件 | SSH Key 认证正常，密码认证拒绝，现有管理访问未中断。 |

### Step 5：配置 UFW

| 项目 | 检查内容 |
| --- | --- |
| 变更前提 | UFW 备份、腾讯云控制台访问、带外恢复能力及批准 SSH 来源 CIDR 均已确认；现有管理员 Key 会话保持可用。 |
| 执行检查 | [ ] 先确认来源受限的 SSH/22 访问规则，再收紧默认入站策略。<br>[ ] 仅按批准范围保留 TCP `22`、`80`、`443`；SSH 不得向全网开放。<br>[ ] 不因规则调整部署、启动或暴露未批准服务。 |
| 验证 | [ ] **22/80/443 正常**：批准来源的 SSH/22 可达且管理员 Key 可登录；80/443 规则与批准范围一致。<br>[ ] 默认策略、编号规则、IPv4/IPv6 状态及实际监听状态均已复核。 |
| 放行条件 | 无未批准规则或端口，SSH 管理路径可用，云侧边界审阅结论一致。 |

### Step 6：开启日志审计

| 项目 | 检查内容 |
| --- | --- |
| 变更前提 | `security-config-backup` 已通过；时间源、日志留存、审计范围和容量阈值已获批准。 |
| 执行检查 | [ ] 按批准方案启用时间同步、持久日志与系统审计。<br>[ ] 审计覆盖 SSH、sudo、账号/组、UFW 与安全配置变更。<br>[ ] 不以关闭日志或审计作为故障规避手段。 |
| 验证 | [ ] **日志正常**：时间同步健康；认证、sudo、UFW 与审计事件可检索；持久化与轮转状态符合批准方案。 |
| 放行条件 | 日志/审计服务健康、事件可检索、容量与留存状态已记录。 |
