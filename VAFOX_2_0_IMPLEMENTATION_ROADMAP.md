# VAFOX 2.0 企业级 AI 操作系统实施路线图

## 0. 文档目标与执行边界

本文档基于 `VAFOX_2_0_PRODUCTION_STANDARD.md` 制定 VAFOX 2.0 企业级 AI 操作系统实施路线图，用于指导 VAFOX 从 1.0 实验系统升级为企业级 AI 操作系统。

本路线图只输出实施规划、阶段目标、建设内容、交付物、依赖关系、风险控制与验收标准；不包含、不要求、也不授权任何服务器登录、部署、变更、重启、数据修改或生产操作。

当前架构定位如下：

| 域名 | VAFOX 2.0 定位 | 核心职责 |
| --- | --- | --- |
| `gateway.vafox.com` | Unified Identity Gateway | 统一入口、身份认证、用户管理、权限体系、单点登录、安全审计入口 |
| `huyan.vafox.com` | CEO Control Plane | CEO AI、CEO 驾驶舱、Robot Control、Approval Center、Knowledge Brain |
| `ai.vafox.com` | AI Workforce Platform | 员工 AI 入口、Agent 平台、工作流、知识库、AI 员工协作 |
| `core.vafox.com` | Enterprise Data Brain | SAP 同步、数据服务、数据治理、数据分析、企业事实与知识中枢 |

## 一、VAFOX 2.0 总体目标

### 1.1 升级定义

VAFOX 2.0 的总体目标，是将 VAFOX 1.0 从实验性 AI 系统升级为企业级 AI 操作系统，使其具备统一身份、分层控制、智能执行、数据中枢、自愈运维和可扩展 Agent 生态能力。

升级后的 VAFOX 2.0 不只是若干 AI 应用的集合，而是面向企业经营、员工协作、机器人执行、数据治理和生产运维的统一 AI Operating System。

### 1.2 总体建设原则

- 统一入口：所有用户、CEO、员工、Robot、Agent 通过 `gateway.vafox.com` 进入系统。
- 分层控制：`huyan.vafox.com` 作为 CEO Control Plane，承担经营控制、审批、指令和验收闭环。
- 智能执行：`ai.vafox.com` 作为 AI Workforce Platform，承载 AI 员工、Agent 编排、工作流和任务执行。
- 数据中枢：`core.vafox.com` 作为 Enterprise Data Brain，提供可信数据、知识、指标、事实和数据服务。
- 标准发布：生产变更必须遵循 Artifact、Checksum、Installer、Verify 发布闭环。
- 最小权限：CEO、员工、Robot、Agent、执行用户权限边界清晰，默认无权限，按职责授权。
- 全链路审计：身份、审批、任务、数据访问、Agent 动作、发布和运维操作必须可追溯。
- 可恢复生产：数据库、知识库、配置、工作流和关键资产必须具备备份、恢复和验证能力。

### 1.3 目标状态

VAFOX 2.0 完成后应达到以下目标状态：

1. CEO 可以通过 Huyan 查看企业经营状态、发起指令、审批高风险动作、验收 AI 员工成果。
2. 员工可以通过统一身份入口访问岗位授权范围内的 AI 能力、任务流和知识库。
3. Robot 与 Agent 可以在受控身份、受控权限、受控审计下执行自动化任务。
4. Core 提供统一数据口径、统一数据服务和可信知识资产，避免数据孤岛与口径分裂。
5. 生产发布、配置、Secret、日志、备份、安全和扩展遵循统一标准。
6. 系统具备监控、自动诊断、自动修复、自动回滚的 Self-Healing Operations 能力。

## 二、Phase 0：标准冻结

### 2.1 阶段目标

Phase 0 的目标是在执行任何实质性系统建设前，冻结 VAFOX 2.0 的企业生产标准，避免边建设边改变方向，确保后续所有阶段围绕统一架构、统一身份、统一发布和统一 Agent 接入标准推进。

### 2.2 架构标准冻结

需要冻结以下架构标准：

- 四层架构边界：Gateway、Huyan、AI、Core 的职责不可混淆。
- 标准访问链路：用户、CEO、员工、Robot、Agent 均通过 Gateway 进入。
- 数据访问原则：业务应用与 Agent 通过 Core 获取可信数据，不绕过 Core 直接读取未授权数据源。
- 控制面原则：高风险动作必须经过 Huyan 的审批、授权与审计链路。
- 执行面原则：AI Workforce 负责任务执行，但不得绕过 Huyan 与 Gateway 的权限控制。
- 扩展原则：GPU、BI、新业务 Agent、新 Robot 均按注册、授权、审计、备份、安全标准接入。

### 2.3 身份标准冻结

需要冻结以下身份标准：

- CEO 用户：最高业务决策身份，但不等同于系统 Root。
- 员工用户：按岗位、组织、职责、数据范围授权。
- Robot 用户：用于 Agent、自动化任务、系统集成的运行身份。
- `vafox-exec` 用户：用于受控执行发布、启动、验证和标准运维动作。
- 禁止共享账号：CEO、员工、Robot 不得共用身份。
- 身份审计：所有登录、授权、权限变更、Token 签发、Robot 调用必须有审计记录。

### 2.4 Release 标准冻结

需要冻结以下 Release 标准：

- Artifact 是唯一允许进入生产的发布载体。
- 每个 Artifact 必须绑定版本、Git Commit、构建时间、构建环境说明和 Checksum。
- Installer 是生产安装唯一入口，必须具备预检查、安装、失败退出、版本记录和回滚说明。
- Verify 是发布完成后的必选动作，至少覆盖服务状态、健康检查、版本检查、关键 API 冒烟、依赖连接、日志错误扫描和回滚条件确认。
- 禁止在生产服务器直接修改代码、直接拉取未验收分支、跳过校验重启生产服务、携带测试 Secret 进入生产。

### 2.5 Agent 标准冻结

需要冻结以下 Agent 标准：

- Agent 必须注册 Agent ID、名称、业务域、负责人、运行身份、版本、上线时间、停用方式。
- Agent 必须声明能力边界，包括任务类型、可访问系统、可读数据、可写数据、外部调用和失败处理。
- Agent 权限默认无权限，必须按任务授权。
- 高风险 Agent 动作必须进入审批流程。
- Agent 每次生产动作必须记录触发来源、输入摘要、执行动作、影响对象、输出结果、状态、错误、重试和审批依据。

### 2.6 关键交付物

- VAFOX 2.0 架构边界冻结清单。
- VAFOX 2.0 身份与权限模型冻结清单。
- VAFOX 2.0 Release 流程冻结清单。
- VAFOX 2.0 Agent 注册与审计标准冻结清单。
- 后续 Phase 1 至 Phase 6 的变更准入原则。

### 2.7 Phase 0 验收标准

Phase 0 完成条件：所有后续建设均可明确映射到 Gateway、Huyan、AI、Core 四层架构，并且身份、发布、Agent、审计、安全标准不存在未决策的关键空白。

## 三、Phase 1：Huyan CEO Control Plane

### 3.1 阶段目标

建设 `huyan.vafox.com` 作为 VAFOX 2.0 的 CEO Control Plane，使 CEO 能够通过统一控制面完成经营洞察、AI 问答、Robot 管控、审批处理、知识检索和生产动作验收。

### 3.2 CEO AI

建设内容：

- CEO 专属 AI 助手，面向经营分析、业务诊断、风险提示和战略问答。
- 接入 Core 的可信数据服务、指标口径和知识资产。
- 支持 CEO 视角的问题追问、摘要生成、异常解释和决策建议。
- 所有回答标注数据来源、生成时间、置信程度和必要的审计信息。

交付物：

- CEO AI 入口设计。
- CEO AI 问答场景清单。
- CEO AI 数据来源与权限边界。
- CEO AI 审计记录规范。

### 3.3 CEO 驾驶舱

建设内容：

- 企业经营总览，包括销售、库存、利润、现金流、供应链、客户、门店或业务单元状态。
- 异常预警、趋势分析、关键指标解释和行动建议。
- 指标口径从 Core 获取，避免 Huyan 自行维护业务事实。
- 支持按时间、组织、区域、业务域、风险等级下钻。

交付物：

- CEO Dashboard 指标目录。
- 指标口径与 Core 数据服务映射表。
- 经营异常规则清单。
- 驾驶舱验收样例。

### 3.4 Robot Control

建设内容：

- Robot 与 Agent 状态总览。
- Agent 启用、停用、暂停、恢复、重试、任务查询和风险阻断入口。
- Robot 任务执行日志、失败原因、重试记录和审批关联查看。
- 高风险 Robot 动作必须转入 Approval Center。

交付物：

- Robot Control 控制台设计。
- Robot 状态模型。
- Robot 动作风险分级表。
- Robot 审计查询页面规划。

### 3.5 Approval Center

建设内容：

- 高风险生产动作审批，包括权限提升、数据批量修改、数据导出、生产发布、Agent 上线、核心服务停用等。
- 支持提交、审批、驳回、撤回、转交、超时提醒和审批结果审计。
- 审批单必须关联触发身份、业务理由、影响范围、回滚方案和执行结果。
- CEO、授权管理者和审计角色权限分离。

交付物：

- 审批流分类与状态机。
- 高风险动作目录。
- 审批单字段规范。
- 审批审计报表规划。

### 3.6 Knowledge Brain

建设内容：

- CEO 视角企业知识入口。
- 接入 Core 的知识库、业务事实库、制度文档、历史决策和 Agent 执行经验。
- 支持权限过滤、来源追溯、版本追溯和知识更新流程。
- 高敏知识按 CEO、管理层、员工、Robot 分级授权。

交付物：

- CEO Knowledge Brain 信息架构。
- 知识来源清单。
- 知识权限标签标准。
- 知识问答审计规范。

### 3.7 Phase 1 验收标准

Phase 1 完成条件：CEO 可以在 Huyan 中查看经营驾驶舱、使用 CEO AI、管理 Robot 状态、处理审批、检索企业知识，并且所有高风险动作均进入审计链路。

## 四、Phase 2：Gateway 统一入口

### 4.1 阶段目标

建设 `gateway.vafox.com` 作为 VAFOX 2.0 的 Unified Identity Gateway，统一承载身份认证、用户管理、权限体系、单点登录、安全入口和审计入口。

### 4.2 身份认证

建设内容：

- 统一登录入口。
- 支持 CEO、员工、Robot、系统集成身份。
- 支持强密码、多因素认证、Token 生命周期和登录风险检测。
- 身份认证结果向 Huyan、AI、Core 提供可信身份上下文。

交付物：

- 身份认证流程图。
- 登录与 Token 策略。
- 认证失败与锁定策略。
- 身份审计字段规范。

### 4.3 用户管理

建设内容：

- 用户生命周期管理，包括创建、启用、禁用、离职、角色变更和权限回收。
- 组织、部门、岗位、门店、业务域等基础身份维度。
- Robot 用户与员工用户分离管理。
- 用户变更必须可审批、可追溯、可回滚。

交付物：

- 用户模型。
- 组织与岗位模型。
- Robot 用户模型。
- 用户生命周期流程。

### 4.4 权限体系

建设内容：

- 建立 RBAC 与 ABAC 结合的权限体系。
- 权限维度覆盖系统、模块、功能、数据范围、动作类型和风险等级。
- 权限按最小权限授予，支持临时授权与到期回收。
- 权限变更进入审批与审计流程。

交付物：

- 角色权限矩阵。
- 数据权限模型。
- Robot 权限模型。
- 权限审计报表规划。

### 4.5 单点登录

建设内容：

- Huyan、AI、Core 管理入口统一接入 Gateway SSO。
- 用户登录后基于身份上下文访问授权系统。
- 服务间调用必须携带受信任身份或服务凭证。
- 登出、Token 过期、权限变更必须同步影响下游系统。

交付物：

- SSO 接入规范。
- 下游系统身份上下文标准。
- Token 刷新与吊销策略。
- SSO 验收用例。

### 4.6 Phase 2 验收标准

Phase 2 完成条件：CEO、员工、Robot 均通过 Gateway 完成统一认证和授权，Huyan、AI、Core 不再各自维护割裂的登录入口和权限模型。

## 五、Phase 3：AI Workforce Platform

### 5.1 阶段目标

建设 `ai.vafox.com` 作为 VAFOX 2.0 的 AI Workforce Platform，为员工和企业 Agent 提供统一 AI 入口、Agent 平台、工作流编排和知识库协作能力。

### 5.2 员工 AI 入口

建设内容：

- 员工使用统一身份进入 AI 工作台。
- 支持岗位相关 AI 助手、任务提交、结果查看、反馈与验收。
- 按员工权限过滤数据、知识、工具和可执行动作。
- AI 输出需要具备可解释、可追溯和可反馈机制。

交付物：

- 员工 AI 工作台设计。
- 员工 AI 场景清单。
- 岗位权限映射。
- 员工反馈闭环规划。

### 5.3 Agent 平台

建设内容：

- Agent 注册、能力声明、版本管理、启停管理和状态监控。
- Agent 任务队列、执行日志、失败重试、人工接管和停用机制。
- Agent 权限与 Gateway 身份体系、Huyan 审批体系打通。
- 生产 Agent 不允许以散落脚本形式绕过平台执行。

交付物：

- Agent Registry 模型。
- Agent Capability Schema。
- Agent 生命周期流程。
- Agent 审计日志标准。

### 5.4 工作流

建设内容：

- 支持人工任务、AI 任务、Robot 任务、审批任务和系统集成任务混合编排。
- 支持触发条件、分支、重试、超时、补偿、回滚和告警。
- 高风险工作流节点必须调用 Huyan Approval Center。
- 工作流定义必须可导出、可恢复、可比对、可审计。

交付物：

- 工作流 DSL 或配置标准。
- 工作流状态机。
- 工作流失败处理策略。
- 工作流审计报表规划。

### 5.5 知识库

建设内容：

- 为员工 AI 和 Agent 提供知识检索、RAG、文档问答和业务规则引用能力。
- 知识来源、版本、权限标签和更新时间必须可追溯。
- 原始文档、清洗文本、向量索引、元数据需要备份和重建能力。
- 敏感知识必须按岗位、业务域、Robot 权限过滤。

交付物：

- AI Workforce 知识库接入规范。
- 知识权限标签模型。
- 知识索引重建策略。
- 知识使用审计规范。

### 5.6 Phase 3 验收标准

Phase 3 完成条件：员工可以通过 AI Workforce 使用岗位授权的 AI 能力；Agent 可以通过平台注册、执行、审计、停用；工作流可以支撑人工、AI、Robot 与审批协同。

## 六、Phase 4：Core Data Brain

### 6.1 阶段目标

建设 `core.vafox.com` 作为 VAFOX 2.0 的 Enterprise Data Brain，为 Huyan、AI、Gateway 和 Agent 提供可信数据、知识、指标、事实、治理和分析服务。

### 6.2 SAP 同步

建设内容：

- 建立 SAP 与 Core 的数据同步机制。
- 明确同步对象、字段映射、同步频率、增量策略和失败补偿。
- 同步数据进入统一数据治理流程。
- SAP 数据访问权限和同步日志必须可审计。

交付物：

- SAP 数据对象清单。
- 字段映射表。
- 同步策略设计。
- 同步失败与补偿策略。

### 6.3 数据服务

建设内容：

- 为 Huyan、AI、Agent、BI 提供统一数据服务 API。
- 数据服务输出统一指标口径、业务事实、维度定义和权限过滤结果。
- 数据访问必须记录调用方、身份、接口、参数摘要、结果状态和耗时。
- 禁止下游系统绕过 Core 建立未受控数据副本。

交付物：

- Unified Data Service API 目录。
- 指标与事实模型。
- 数据服务权限策略。
- 数据服务审计规范。

### 6.4 数据治理

建设内容：

- 建立数据质量规则、数据血缘、指标口径、主数据管理和数据生命周期策略。
- 敏感数据分级分类，支持脱敏、加密、访问控制和导出审批。
- 数据删除必须经过审批、备份和可恢复方案。
- 关键数据变更必须有版本、影响评估和复核记录。

交付物：

- 数据治理规则清单。
- 数据质量检查标准。
- 敏感数据分级模型。
- 数据生命周期策略。

### 6.5 数据分析

建设内容：

- 面向 CEO 驾驶舱、业务分析、异常检测和 Agent 决策提供分析能力。
- 统一指标计算口径，避免多系统指标不一致。
- 支持趋势分析、异常识别、预测分析和经营归因。
- 分析结果必须标注数据来源、更新时间和适用范围。

交付物：

- 分析主题域清单。
- 指标计算模型。
- 异常分析规则。
- 分析结果验收样例。

### 6.6 Phase 4 验收标准

Phase 4 完成条件：Core 可以稳定向 Huyan、AI、Agent 提供统一数据服务、可信知识、指标口径和分析结果，并且数据同步、治理、访问、备份、恢复和审计链路完整。

## 七、Phase 5：Self-Healing Operations

### 7.1 阶段目标

建设 VAFOX 2.0 的 Self-Healing Operations 能力，使系统具备监控、自动诊断、自动修复和自动回滚的生产自愈能力，同时确保自愈动作受权限、审批和审计约束。

### 7.2 监控

建设内容：

- 覆盖 Gateway、Huyan、AI、Core 的服务状态、健康检查、API 延迟、错误率、资源使用、任务队列、数据库连接、知识库索引和同步状态。
- 建立业务指标监控，包括审批积压、Agent 失败率、数据同步延迟、关键任务超时。
- 告警按等级路由到 CEO、运维、业务负责人或 Robot Control。
- 监控日志不得输出明文 Secret。

交付物：

- 监控指标目录。
- 告警等级与路由策略。
- 服务健康模型。
- 业务健康模型。

### 7.3 自动诊断

建设内容：

- 对服务异常、接口失败、任务失败、数据同步失败、Agent 异常和工作流阻塞进行自动诊断。
- 诊断结果包含异常类型、影响范围、可能原因、推荐处理、是否需要审批。
- 自动诊断结果进入 Huyan 或运维控制面供人工复核。
- 诊断模型必须可解释、可追溯、可审计。

交付物：

- 自动诊断规则库。
- 异常分类模型。
- 诊断报告模板。
- 诊断审计规范。

### 7.4 自动修复

建设内容：

- 对低风险、标准化、可回滚问题执行自动修复。
- 修复动作包括任务重试、缓存刷新、索引重建、连接恢复、队列重放、配置回退等。
- 高风险修复必须进入 Approval Center。
- 自动修复必须记录触发条件、执行步骤、影响对象、结果和回滚方式。

交付物：

- 自动修复动作白名单。
- 修复风险分级表。
- 修复执行审计规范。
- 人工接管流程。

### 7.5 自动回滚

建设内容：

- 对发布失败、健康检查失败、关键 API 冒烟失败、错误率异常和数据校验失败触发回滚策略。
- 回滚动作必须基于可追溯 Artifact、版本记录、配置备份和数据保护方案。
- 涉及数据结构或数据内容的回滚必须经过更严格审批和恢复验证。
- 回滚完成后必须执行 Verify 与事故复盘。

交付物：

- 回滚触发条件。
- 回滚决策树。
- 回滚验收清单。
- 事故复盘模板。

### 7.6 Phase 5 验收标准

Phase 5 完成条件：VAFOX 2.0 可以发现关键异常、生成诊断结论、对低风险问题执行自动修复、对发布或运行异常执行受控回滚，并且所有自愈动作均可审计。

## 八、Phase 6：Agent Ecosystem

### 8.1 阶段目标

建设 VAFOX 2.0 的企业 Agent 生态，使补货、财务、CRM、供应链、运维、HR 等专业 Agent 可以按统一标准接入、授权、运行、审计、停用和扩展。

### 8.2 补货 Agent

规划内容：

- 分析库存、销量、季节性、供应周期和缺货风险。
- 生成补货建议、采购建议或门店调拨建议。
- 高风险采购、批量调拨、异常供应商选择必须审批。
- 动作结果回写前必须经过权限与数据校验。

### 8.3 财务 Agent

规划内容：

- 支持对账、发票、成本、利润、预算、现金流和异常费用分析。
- 涉及付款、账户、发票、财务凭证、利润敏感数据时启用更高安全等级。
- 财务高风险动作必须人工审批。
- 财务数据访问必须强审计、强权限、强脱敏。

### 8.4 CRM Agent

规划内容：

- 支持客户画像、客户分层、营销建议、客户流失预警和服务跟进。
- 客户隐私数据必须分级授权、脱敏展示和访问审计。
- 批量触达客户、导出客户数据、修改客户等级必须审批。
- CRM Agent 的建议必须支持人工确认与效果反馈。

### 8.5 供应链 Agent

规划内容：

- 支持采购、库存、供应商协同、履约、物流异常和供应风险分析。
- 关键采购、调价、供应商变更必须审批。
- 供应商数据、合同数据和成本数据必须按权限访问。
- 供应链 Agent 需要与 Core 数据服务和 Huyan 审批中心联动。

### 8.6 运维 Agent

规划内容：

- 支持监控分析、异常诊断、任务重试、健康检查和标准化修复建议。
- 低风险修复动作可自动执行，高风险动作必须审批。
- 不得绕过 Release 标准直接修改生产代码或配置。
- 运维 Agent 的执行范围必须与 `vafox-exec` 标准职责严格区分。

### 8.7 HR Agent

规划内容：

- 支持员工入职、离职、权限申请、组织调整、培训问答和绩效辅助分析。
- HR 敏感数据必须按岗位、组织和管理关系授权。
- 员工权限开通、权限提升、离职回收必须进入审批与审计流程。
- HR Agent 不得访问与任务无关的个人敏感信息。

### 8.8 Agent 生态共通交付物

- Agent Catalog。
- Agent 能力声明模板。
- Agent 风险分级模型。
- Agent 权限矩阵。
- Agent 审批与审计规范。
- Agent 下线与应急停用流程。

### 8.9 Phase 6 验收标准

Phase 6 完成条件：补货、财务、CRM、供应链、运维、HR Agent 均具备清晰规划、注册标准、权限边界、审批策略、审计要求和停用机制。

## 九、安全和权限策略

### 9.1 CEO 数据隔离

- CEO 可以查看企业全局经营视图，但不直接获得数据库 Root 权限。
- CEO 数据视图必须通过 Huyan 和 Core 的授权数据服务提供。
- CEO 专属数据、决策记录、审批记录和敏感经营分析必须独立权限控制。
- CEO AI 问答记录按高敏信息保存、检索和审计。
- CEO 高风险动作必须保留审批、授权、执行和结果验收记录。

### 9.2 员工权限

- 员工权限按岗位、部门、业务域、数据范围和动作类型授予。
- 员工不得访问岗位无关数据，不得使用共享账号。
- 员工 AI 入口必须按权限过滤知识、数据、工具和可执行动作。
- 员工权限变更必须有申请、审批、执行、回收和审计记录。
- 离职、转岗、临时授权到期必须触发权限回收。

### 9.3 Robot 权限

- Robot 用户必须与 CEO、员工个人账号分离。
- Robot 默认无权限，按 Agent、任务、系统、数据范围和动作类型授权。
- Robot 不得默认拥有管理员权限，不得保存超出任务需要的敏感数据。
- Robot 高风险动作必须进入 Approval Center。
- Robot 每次动作必须记录 Agent ID、运行身份、触发来源、输入摘要、影响对象、执行结果和审批依据。

### 9.4 审批流程

- 审批流程覆盖权限提升、数据批量修改、数据批量导出、生产发布、Agent 上线、核心服务停用、财务动作、供应商变更等高风险场景。
- 审批单必须包含业务理由、风险等级、影响范围、执行计划、回滚方案、申请人、审批人、执行人和验收人。
- 审批权限与执行权限分离，避免单人完成高风险闭环。
- 审批结果必须同步给 Huyan、AI、Gateway、Core 相关执行或访问控制模块。
- 审批记录不得被普通用户、Robot 或 Agent 修改。

### 9.5 安全基线

- 强制身份认证。
- 最小权限授权。
- Secret 加密管理与轮换。
- 全链路审计。
- 高风险动作阻断。
- 生产发布可追溯。
- 数据备份可恢复。
- 未审批修改生产、直接删除数据、绕过审批必须被制度与技术控制禁止。

## 十、迁移策略

### 10.1 Legacy 冻结

- 冻结 VAFOX 1.0 的新增能力开发，只允许必要缺陷修复、安全修复和数据保护动作。
- 梳理 Legacy 系统、脚本、账号、数据源、任务、Agent、配置和依赖。
- 标记需要保留、迁移、重构、废弃的组件。
- Legacy 变更必须记录原因、影响范围和回退方式。

### 10.2 数据保护

- 在迁移前完成关键数据库、知识库、配置、工作流和审计日志备份。
- 明确迁移数据范围、字段映射、权限标签和质量校验规则。
- 对敏感数据执行脱敏、加密、访问控制和导出审批。
- 建立迁移前后数据校验，包括数量、字段、口径、权限和业务样例验证。

### 10.3 灰度迁移

- 按用户、业务域、Agent、数据服务或功能模块分批迁移。
- 优先迁移低风险、只读、可回滚场景。
- 每批灰度迁移必须有验收指标、回滚条件和观察窗口。
- 灰度期间保留 Legacy 与 VAFOX 2.0 的对照验证机制。

### 10.4 验证

- 验证身份登录、权限过滤、数据口径、审批流程、Agent 执行、知识检索、工作流和审计记录。
- 验证发布链路是否满足 Artifact、Checksum、Installer、Verify 标准。
- 验证备份恢复、异常诊断、自动修复和回滚流程。
- 验证 CEO、员工、Robot 的典型业务路径。

### 10.5 切换

- 只有当灰度迁移满足验收标准后，才允许进入正式切换决策。
- 切换必须具备明确窗口、负责人、审批记录、回滚方案和业务通知。
- 切换后进入稳定观察期，持续监控服务健康、业务指标、数据同步和 Agent 失败率。
- Legacy 系统下线必须在数据保护、审计留存、回滚窗口结束后执行规划，不得直接删除生产数据。

## 十一、最终验收标准

### 11.1 VAFOX 2.0 完成定义

当且仅当以下条件全部满足时，VAFOX 2.0 可以判定为企业级 AI 操作系统实施完成：

1. `gateway.vafox.com`、`huyan.vafox.com`、`ai.vafox.com`、`core.vafox.com` 四层架构角色边界清晰，并按统一入口、分层控制、智能执行、数据中枢模式运行。
2. Gateway 已承载统一身份认证、用户管理、权限体系和单点登录能力。
3. Huyan 已承载 CEO AI、CEO 驾驶舱、Robot Control、Approval Center 和 Knowledge Brain。
4. AI Workforce 已承载员工 AI 入口、Agent 平台、工作流和知识库协作能力。
5. Core 已承载 SAP 同步、数据服务、数据治理和数据分析能力。
6. 所有生产发布均遵循 Artifact、Checksum、Installer、Verify 标准。
7. CEO、员工、Robot、`vafox-exec` 用户权限边界清晰，且权限变更可审批、可追溯、可回收。
8. 所有生产 Agent 完成注册、能力声明、最小权限授权、审计记录和停用机制。
9. 数据库、知识库、配置、工作流和审计资产具备备份、恢复、校验和验收记录。
10. 高风险动作必须进入审批流程，未审批修改生产、直接删除数据、绕过审批被制度和技术控制禁止。
11. Self-Healing Operations 具备监控、自动诊断、自动修复和自动回滚规划与验收机制。
12. 补货、财务、CRM、供应链、运维、HR Agent 生态具备可扩展接入标准。
13. Legacy 迁移完成数据保护、灰度迁移、验证、切换和稳定观察。
14. CEO 可以通过 Huyan 查看、审批、指挥、验收 VAFOX 2.0 的关键生产动作。
15. 每一次身份访问、数据访问、Agent 动作、审批动作、生产发布和运维动作均可审计、可追溯、可回滚或可解释。

### 11.2 最终验收交付物

- VAFOX 2.0 架构验收报告。
- Gateway 统一身份与权限验收报告。
- Huyan CEO Control Plane 验收报告。
- AI Workforce Platform 验收报告。
- Core Data Brain 验收报告。
- Self-Healing Operations 验收报告。
- Agent Ecosystem 接入标准与首批 Agent 规划报告。
- 安全、权限、审批、审计验收报告。
- Legacy 迁移与切换验收报告。
- VAFOX 2.0 企业级 AI 操作系统最终验收结论。

### 11.3 明确非目标

本路线图不包含任何实际服务器操作，不要求登录任何服务器，不要求修改生产配置，不要求重启服务，不要求迁移真实数据，不要求执行发布命令。所有实际生产动作必须在后续获得明确授权、审批和执行窗口后，另行按照 VAFOX 2.0 生产标准执行。
