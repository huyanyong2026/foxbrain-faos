# FireFox Explorer 手机号登录升级报告

## 完成内容

- Explorer 日常入口由微信授权调整为“手机号 + 短信验证码”。
- Gateway 页头和页脚使用正式火狐狸品牌 Logo，替换原红色字母标识，并适配手机尺寸。
- 保留原有 Explorer 身份、购买记录和探索数据，旧数据结构兼容保留。
- 新增一次性短信验证码：五分钟有效、验证后立即失效。
- 同一手机号六十秒内不得重复发送；每小时限制手机号和来源请求次数。
- 连续输错达到五次后锁定当前验证码，必须重新获取。
- 验证成功后才允许通过 Core 只读接口匹配本人购买记录。
- 未配置正式短信通道时明确提示暂未开放，不显示或记录测试验证码。

## 数据与隐私

- 新增 `explorer_phone_codes`，只保存手机号、验证码和来源地址的 HMAC 摘要。
- 完整手机号、短信验证码、Core 令牌和短信服务令牌均不写入业务日志或 Git。
- Explorer 只能查看自己的身份、装备和探索记录。
- 未增加 SAP 连接、SAP 写权限或 SAP 修改操作。

## 页面与接口

- `GET /explorer/register`：手机号登录、短信验证码及隐私同意。
- `POST /api/explorer/phone/code`：发送一次性短信验证码。
- `POST /explorer/auth/phone`：验证手机号并创建或登录 Explorer。
- 原微信回调地址仅重定向到新的手机号登录页，不再承担认证。

## 短信服务配置

生产环境通过独立权限文件配置：

- `EXPLORER_SMS_ENDPOINT`：可信 HTTPS 短信 Webhook。
- `EXPLORER_SMS_TOKEN`：Webhook Bearer 令牌。

Webhook 接收 `phone`、`code`、`scene`、`expires_minutes`，成功时返回 HTTP 2xx。两项配置任一缺失时，发送功能保持关闭。

## 验证范围

- 手机号创建身份与再次登录。
- 短信验证码单次使用、过期、重发限制、频率限制和错误次数锁定。
- 数据库不保存明文手机号和验证码。
- 登录后只显示本人的 Core 购买匹配结果。
- 未登录、跨站请求、非允许方法及旧微信回调防护。
- 未配置短信服务时安全关闭。

## 上线边界

代码可在不启用短信通道的情况下部署。正式开放登录前，只需在服务器配置通过安全审核的短信 Webhook 地址和令牌，然后重启 Explorer 身份服务；不需要修改 SAP 或 Core 数据。
