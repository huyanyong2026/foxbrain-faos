# 重复页面与旧入口

## 结论

生产应用仍保留大量历代首页、Dashboard、工作台和技术入口。新版五入口负责日常导航，但旧路由没有删除，属于“导航隐藏但代码仍在”。

## 代表性重复入口

- 首页类：`/`、`/ceo-home`、`/ceo-workbench`、`/overview`、`/boss`、`/desktop`、`/portal`。
- 经营类：`/business-overview`、`/business-radar`、`/command-center`、`/ai-operations`。
- AI 类：`/copilot`、`/jarvis`、`/ai-assistant`、`/ai-ceo`、`/digital-brain`、`/agents`。
- 决策类：`/decision`、`/decision-center`、`/decisions`、`/inventory-decision`。
- 图谱类：`/knowledge-graph`、`/enterprise-knowledge-graph`、`/graph`、`/entity-center`。
- 知识类：`/knowledge`、`/knowledge/dashboard`、`/knowledge-pipeline`、`/owner/knowledge`、`/second-brain`。

## 结构风险

- 约 2.5 万行的单文件同时承载页面、API、数据库初始化和业务规则。
- 大量早期 API 仍可被路由分发，维护者难以判断哪个是正式版本。
- 部分旧页面仍含英文技术词或乱码片段。
- 多个模块 README 只是架构占位，不能等同于真实功能。

本次仅登记，未隐藏、删除或合并任何入口。
