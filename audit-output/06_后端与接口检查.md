# 后端与接口检查

## 架构现状

- 使用 Python `BaseHTTPRequestHandler` 单进程服务。
- 页面、API、数据初始化、任务、AI、同步和权限逻辑集中于一个约 2.5 万行文件。
- SQLite 每次请求独立连接，无显式连接池。
- 生产进程工作目录和启动文件均指向 `/opt/firefox-portal`。

## 正向控制

- SQL 大部分使用参数绑定。
- 密码使用 PBKDF2-SHA256，支持旧迭代次数升级。
- 登录失败有锁定机制。
- Cookie 设置 HttpOnly、Secure、SameSite=Lax。
- 管理和敏感模块存在角色检查。

## 主要问题

- 未发现 CSRF token；大量 POST 接口依赖 SameSite Cookie，仍不足以替代完整 CSRF 防护。
- 会话 Cookie 只签名用户 ID，没有明确服务端会话、过期时间、撤销表或设备管理。
- 动态表名 SQL 依赖代码白名单正确性，维护风险高。
- 异常日志近 7 天出现 21 次 Traceback，需逐条归因。
- 生产源码与 Git commit 不一致，接口行为无法由仓库提交精确复现。
