# 性能与运维检查

## 资源现状

- 根磁盘 217 GB，已用约 41 GB（20%）。
- 内存 15 GB，可用约 11 GB。
- Swap 约 10 GB，使用很少。
- SQLite 数据库约 749 MB。
- 上传目录约 4.5 MB。
- 本机未登录 GET 响应约 1-4 ms。

## 运维优点

- Web 服务由 systemd 管理并 active。
- Nginx 反向代理到本机 8088。
- 存在每日备份 timer。
- SAP 副本数据库只监听本机 15432。

## 风险

- 单进程 `BaseHTTPRequestHandler`，监听 backlog 为 5，不适合并发员工规模扩大。
- 749 MB SQLite 同时承载 216 张表，写并发、锁等待和备份恢复风险上升。
- 生产版本由直传文件构成，服务器 Git 仓库落后，发布和回滚不可复现。
- 近 7 天 Web 服务日志有 21 次 Traceback。
- SAP 同步存在双调度。
- 多个外围服务端口直接公网监听。
- 未发现完整 APM、慢查询、告警闭环和容量阈值证据。
