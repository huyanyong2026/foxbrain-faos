# 建议修复路线图

本路线图仅提出建议，不生成或应用补丁。

## 第一阶段：生产安全与可追溯（立即）

1. 人工确认 SAP 22:00 唯一调度方式，检查同步脚本互斥与幂等。
2. 盘点生产运行文件与 Git 差异，形成可复现发布版本。
3. 核对公网容器端口和云安全组，关闭非必要公网入口。
4. 对 21 次 Traceback 分类，确认是否影响数据或泄露敏感信息。

## 第二阶段：数据基础（1-2 个 Sprint）

1. 建立六类标准主档的映射、人工确认和来源追踪。
2. 设计外键或应用级一致性检查，不直接在生产迁移。
3. 建立 SAP 读取数、暂存数、发布数、业务表数的统一对账。
4. 连续观察至少 7 天同步结果和数据新鲜度。

## 第三阶段：知识与 AI 真实验收（1-2 个 Sprint）

1. 用非敏感样本验收上传、哈希、提取、OCR、分块、摘要、对象关联、权限、检索和引用。
2. 验证删除、版本、重新索引和权限变更同步。
3. 八个智能体逐一验证数据源、工具、审批、证据和越权阻断。

## 第四阶段：员工开放前治理

1. 增加 CSRF、会话过期和服务端撤销。
2. 建立岗位权限矩阵和敏感字段分级。
3. 建立正式路由清单，逐步停用历史入口。
4. 完成并发、备份恢复和灾难恢复演练后再扩大用户。
