## 一、实施目标

### 1.1 第一阶段目标

将 `huyan.vafox.com` 准备为 CEO 的受控智能入口：在统一身份、最小权限、可追溯审计和人工审批约束下，汇聚 CEO Dashboard、受治理的 Agent Hub 与 Knowledge Brain，形成“**洞察 → 建议 → 待审批行动 → 结果留痕**”的闭环。

第一阶段的交付重点是可实施、可验证、可回退的准备包，而不是启用任何生产执行能力。默认能力为读取已授权的经营上下文、生成分析与建议；涉及外部写入、生产发布、资金、主数据或不可逆动作的请求，必须保持阻断或转为人工审批任务。

### 1.2 成功定义

1. CEO 可经 Gateway Identity 完成身份验证、VID/角色解析及到 `huyan.vafox.com` 的受控路由。
2. Dashboard 可呈现具备数据口径、更新时间、权限范围、证据来源和风险等级的 CEO 经营视图。
3. Knowledge Brain 只在调用者获授权的数据范围内提供可溯源检索、摘要、问答与决策记忆。
4. Agent Hub 中每个 Agent 均有唯一身份、负责人、版本、能力边界、数据/工具授权、审计和紧急暂停路径。
5. CEO Control Plane 仅编排已批准的只读查询、建议生成和人工审批工作流；默认不直接控制 SAP、Core、数据库、服务器或云资源。

### 1.3 非目标

- 不替代 SAP、Core 或其数据权威性；SAP 仍是业务事实来源，Core 仍是企业数据与权限执行权威。
- 不建设第二套账号、VID、角色、权限或企业数据库。
- 不把 Huyan 暴露为通用运维跳板、自由脚本执行器或高权限自动化入口。
- 不在第一阶段启用自主执行、跨系统写回、自动发布、自动扩缩容或绕过审批的 Agent。

## 二、Huyan 目标架构

### 2.1 分层架构

```text
CEO
  │  Gateway Identity：认证、VID、角色、权限快照、会话与路由
  ▼
huyan.vafox.com — CEO Private Brain / CEO Control Plane
  ├─ CEO Dashboard：经营健康、简报、风险、机会、决策与行动队列
  ├─ Agent Hub：Registry、编排、策略校验、审批引用、审计与紧急暂停
  ├─ Knowledge Brain：受权限约束的检索、引用、摘要、决策记忆
  └─ Control Plane：只读任务编排、审批工作流、证据与结果归档
       │
       ├─ Core：组织、业务、权限与数据访问权威
       ├─ AI / FoxBrain：分析、推理、模型与受控工具能力
       └─ 企业来源：SAP、数据中心及其他已登记来源（经 Core 边界访问）
```

### 2.2 权威边界与数据流

| 层级 | 责任 | 不得承担的责任 |
| --- | --- | --- |
| Gateway Identity | 验证凭据、解析 VID/角色、发放短时会话、生成路由与权限快照 | 直接授予业务数据权限、承载企业业务数据、替代 Core 授权 |
| Huyan Experience | 组织 CEO 工作台与展示受控结果 | 独立认证、保存原始凭据、绕过权限判断 |
| CEO Control Plane | 校验 Agent 状态/版本/授权/审批；编排只读请求与待审批动作 | 直接执行高风险生产动作、永久持有高权限凭据 |
| Core | 企业数据、组织关系、数据权限及业务边界的权威执行层 | 被 Huyan 复制或绕过 |
| Knowledge Brain | 在获准上下文中检索、引用、总结、形成记忆候选 | 将未授权内容写入提示词、索引或回答 |
| Agent Hub | 注册、生命周期、路由、限额、审计、暂停 | 让未注册或未审批 Agent 获得生产能力 |

### 2.3 核心控制原则

- **零信任与默认拒绝：** 每次敏感访问均验证会话、VID、角色、权限版本、Agent 状态、资源范围和审批有效期。
- **最小权限：** 权限按人、Agent、版本、环境、数据集、工具、动作和时限绑定；CEO 身份不等于无限制系统权限。
- **读写分离：** 第一阶段仅允许已批准的读取、分析、草拟和提交“待审批行动”；写入操作必须由权威系统在独立审批后执行。
- **证据优先：** Dashboard 数字、Knowledge 回答和 Agent 建议必须标识来源、口径、更新时间、权限范围和不确定性。
- **可停用：** Gateway 会话、Agent、工具连接、任务队列和动作模板须具有独立撤销/暂停开关。

## 五、Agent 接入计划

### 5.1 接入范围与顺序

第一阶段按“低风险、只读、可解释”顺序接入，建议候选包括 CEO Briefing、企业健康、风险雷达、机会雷达、决策建议和知识查询 Agent。任何具有写入、外部通信、资金、人员、主数据、部署或基础设施控制能力的 Agent 均不进入第一阶段 Active 范围。

### 5.2 每个 Agent 的注册包

每个候选 Agent 必须在接入前提交：唯一 Agent ID、名称、版本、业务与技术负责人、用途、能力/禁止动作、风险等级、输入输出契约、数据分类与来源、工具清单、权限矩阵、限额、审计字段、测试计划、依赖、紧急暂停和回滚方案。

### 5.3 生命周期门禁

1. **Draft：** 仅设计，无运行身份、数据或工具权限。
2. **Registered：** Agent Hub 完整性校验通过，只能进入隔离测试。
3. **Approved：** 安全、测试、业务、技术和权限审批完成；生产权限仍默认未启用。
4. **Active：** 仅在完成上线门禁后按批准的最小范围服务请求。
5. **Suspended / Deprecated：** 出现异常、审批失效、未授权变更或退役时，立即阻断任务、会话、工具和凭据；保留合规审计证据。

### 5.4 调用控制

- Huyan 调用时必须传递经验证的 VID、活跃角色、权限版本、会话关联 ID、Agent ID/版本、数据范围和审批引用（如需要）。
- Agent Hub 必须在调用前验证 Agent 为 Active、版本匹配、权限未过期、数据范围允许、工具白名单命中、限额未超限。
- Agent 输出必须区分“事实/来源”“推断”“建议”“不确定性”和“待审批动作”；不得将建议伪装成已执行结果。
- 任一校验失败时拒绝调用、记录审计事件，并在高风险场景触发告警或暂停。

## 六、Dashboard 接入计划

### 6.1 目标页面能力

Dashboard 在第一阶段承载：企业健康、CEO 日报、风险雷达、机会雷达、决策中心、AI 建议和待审批行动。每张卡片/图表必须显示数据口径、来源、更新时间、覆盖范围、权限状态和风险/不确定性提示。

### 6.2 接入步骤

1. 定义 CEO 指标字典：指标名称、业务口径、权威来源、刷新频率、负责人、敏感级别及降级行为。
2. 为每个 Dashboard 模块定义只读 API 契约：请求所需权限、返回字段分级、错误码、缓存边界、审计字段和版本策略。
3. 通过 Gateway 会话和 Core 权限上下文执行服务端授权；前端不得承担最终权限判断。
4. 将 Agent/Knowledge 生成的内容标注为“AI 分析”并展示证据引用、生成时间、模型/Agent 版本及人工确认状态。
5. 将行动入口默认连接至“待审批任务”而非直接执行；无有效审批时应明确显示不可执行原因。

### 6.3 验证重点

- CEO 仅看到获授权的组织、门店、品牌、人员及经营范围。
- 数据延迟、来源异常、无权限或服务不可用时，页面不展示误导性的“正常”结论。
- 深链接、浏览器缓存、导出、截图和错误页均不得泄露越权数据或令牌。

## 七、Knowledge Brain 接入计划

### 7.1 知识边界

Knowledge Brain 通过登记的数据连接器访问授权来源；不复制 SAP/核心数据权威，不以未审核文件、个人私密记忆或跨租户内容作为默认语料。知识对象必须包含来源系统、对象 ID、所有者、数据分类、ACL、更新时间、保留期限和引用位置。

### 7.2 接入顺序

1. 建立数据源清单与数据分类：先接入低风险、结构化、权威、可审计的 CEO 所需资料。
2. 定义连接器契约：认证方式、允许字段、同步/检索频率、失败处理、删除/失效传播和审计事件。
3. 建立检索前权限过滤：先按 VID、角色、组织、数据范围与敏感等级裁剪候选集，再进行检索和生成。
4. 建立输出治理：回答附带来源、时间、适用范围、置信度/不确定性；敏感内容遵循脱敏、最小展示和禁止导出策略。
5. 建立决策记忆流程：仅将已确认且可保留的决策、依据、审批与结果写为记忆候选；记忆写入须有数据所有者与保留策略。

### 7.3 安全与质量门禁

- 以权限拒绝测试、跨组织隔离测试、已撤销权限测试、删除传播测试和提示注入测试作为接入必测项。
- 知识回答不能引用调用者无权访问的文档标题、片段、向量特征或元数据。
- 未找到可靠来源时，必须明确说明“无足够证据”，不得补造事实。

## 八、Gateway 接入计划

### 8.1 接入原则

`gateway.vafox.com` 是统一身份入口，负责认证、VID 解析、角色识别、权限快照与路由；Founder/CEO 的受控目标路由为 `https://huyan.vafox.com`。Huyan 不创建二次登录入口、不保存原始凭据、不以客户端角色提示作为最终授权依据。

### 8.2 会话与路由契约

Huyan 接入前须与 Gateway 固化会话契约：签发方、受众、签名算法与密钥轮换、令牌时限、刷新与撤销、`vid`、活跃角色、权限版本、上下文版本、会话关联 ID、目标路由、错误码和降级行为。会话负载只携带最小身份与权限声明；企业详情应在 Huyan 侧经 Core 受控查询获取。

### 8.3 接入步骤

1. 明确 CEO/Founder 角色、委托角色和可访问 Huyan 的最小权限集合。
2. 定义 Gateway → Huyan 的路由允许列表、回调/重定向校验和防重放策略。
3. 在 Huyan 服务端实现令牌签名、签发方、受众、有效期、撤销状态与权限版本校验；任何失败均拒绝访问。
4. 将会话、路由、权限决策和拒绝事件写入统一审计链路，避免记录原始凭据或完整令牌。
5. 验证权限/角色变更后会话失效、登出、令牌过期、重放、跨域与错误重定向行为。

## 九、安全检查清单

以下检查是后续实施前的门禁，不是本阶段的操作指令。

### 9.1 身份、权限与审批

- [ ] Gateway 是唯一的人类身份入口；无 Huyan 本地密码或平行账号体系。
- [ ] 所有 CEO、委托人、服务账号和 Agent 均有独立身份、最小角色和可撤销凭据。
- [ ] 会话令牌短时、签名可验证、受众受限、可撤销；令牌/凭据不进入 URL、浏览器存储、日志或错误报告。
- [ ] 高风险操作有职责分离的人工审批、范围、有效期和审批编号；审批失效即拒绝。
- [ ] Agent 只能使用与 Agent ID、版本、环境和资源绑定的最小权限。

### 9.2 数据、知识与隐私

- [ ] 数据源、字段、分类、所有者、ACL、保留和删除规则已登记。
- [ ] 检索前执行 ACL/组织范围过滤，检索后执行输出脱敏和引用校验。
- [ ] 日志、备份、分析和测试数据不含原始凭据、完整令牌、敏感原文或超范围个人信息。
- [ ] 关键数字均能回溯到权威来源、口径和更新时间；AI 推断与事实明显区分。

### 9.3 应用、接口与运行时

- [ ] 所有入口使用 HTTPS、可信证书、HSTS（经兼容性评估）和安全响应头；不存在调试、管理或未认证 API 暴露。
- [ ] API 实施服务端鉴权、输入校验、输出编码、请求大小/速率限制、超时、重试边界和幂等设计。
- [ ] 依赖、镜像、构建产物和配置均完成版本锁定、漏洞扫描、来源校验与变更审查。
- [ ] Agent 工具为白名单调用；禁止任意 shell、任意 URL、任意文件、任意数据库或任意云控制接口。
- [ ] 安全事件可关联请求 ID、会话 ID、VID（按脱敏规则）、Agent/版本、审批号、策略版本与结果。

### 9.4 运维与恢复

- [ ] 生产、测试、开发环境、凭据、日志和备份相互隔离；密钥可轮换且轮换演练通过。
- [ ] 监控覆盖可用性、授权拒绝率、异常调用、Agent 失败率、知识检索越权、延迟、成本/配额与审计链路健康度。
- [ ] 已验证 Agent 暂停、会话撤销、工具断连、任务队列冻结和版本回退不会破坏审计保留。
- [ ] 任何实际部署均具备独立批准、变更窗口、影响范围、值守、回滚负责人和事后报告。

| 类别 | 验收标准 | 证据 |
| --- | --- | --- |
| 架构 | 权威边界、数据流、接口清单、信任边界与风险模型经业务/技术/安全确认 | 签署的架构评审记录 |
| 身份 | Gateway → Huyan 会话/路由契约冻结；令牌验证、过期、撤销、角色变更与拒绝路径测试通过 | 接口契约、测试报告、审计样本 |
| 权限 | CEO、委托人、服务账号与 Agent 权限矩阵批准；默认拒绝与最小权限测试通过 | 权限矩阵、审批记录、负向测试结果 |
| Agent | 每个首批 Agent 的注册包、生命周期状态、安全审核、限额、暂停及回滚方案齐全 | Registry 条目、审核/测试/审批证据 |

## 十一、回滚方案

### 11.1 回滚原则

回滚的第一目标是恢复安全边界与业务连续性，而非保留新功能。回滚必须优先采用可逆的“关闭入口/撤销授权/切回已验证版本”方式；不得为回滚而删除审计记录、篡改业务数据或执行未经批准的数据库、服务器、Docker 或云操作。

### 11.2 触发条件

- 身份、权限、路由或会话验证发生绕过、越权、重放或大规模异常拒绝。
- Agent 出现未批准工具调用、超范围数据访问、不可解释高风险建议或异常成本/负载。
- Knowledge Brain 出现跨组织/跨角色泄露、错误引用、删除未传播或提示注入影响。
- Dashboard 展示的数据口径错误、过期且未标识，或造成 CEO 决策误导。
- 审计、告警、密钥轮换、紧急停止或审批链路不可用。

### 11.3 分级回退路径（未来执行时使用）

| 级别 | 适用场景 | 回退目标 | 预置证据 |
| --- | --- | --- | --- |
| R1 功能降级 | 单一 Dashboard 卡片、知识源或低风险 Agent 异常 | 关闭受影响功能，保留 Huyan 核心只读体验 | 功能开关清单、影响评估、审计记录 |
| R2 Agent/工具暂停 | Agent、连接器或工具权限异常 | 将 Agent 置为 Suspended，撤销工具令牌，冻结相关任务 | Agent 状态记录、暂停审批/事件号 |
| R3 身份入口保护 | 会话、路由或权限快照异常 | 拒绝 Huyan 新会话/敏感入口，撤销会话，回到 Gateway 安全错误页 | 会话撤销记录、访问审计 |
| R4 应用版本回退 | 已验证版本引入安全或稳定性回归 | 切回经验证的上一应用/配置基线，保持数据与审计完整 | 发布清单、校验和、兼容性确认 |
| R5 全面隔离 | 已知或疑似重大泄露/失控 | 隔离 Huyan 的受影响集成，停止任务分派，启动事件响应 | 事件记录、取证保全、沟通清单 |

### 11.4 回滚完成标准

1. 受影响入口、Agent、会话、工具或连接器已按风险范围停止或撤销。
2. 已恢复至经过验证的版本/配置基线，且未扩大任何权限。
3. Gateway、Dashboard、Agent Hub、Knowledge Brain 与审计链路完成健康与负向权限验证。
4. 事件时间线、影响数据范围、采取动作、遗留风险和恢复审批完成归档。
5. 根因修复、复测及重新审批完成前，不得重新启用受影响能力。

## 十二、未来 AI/Core 连接计划

### 12.1 分阶段路线

| 阶段 | 连接范围 | 允许能力 | 明确限制 |
| --- | --- | --- | --- |
| Phase 1：准备与只读 | Huyan ↔ Gateway、Core/AI 契约设计 | 身份路由、授权查询、只读指标、受控知识检索、AI 建议 | 不部署、不写入、不自动执行 |
| Phase 2：受控读取 | Huyan ↔ Core、Huyan ↔ AI 的已批准 API | CEO 经营视图、风险/机会分析、带引用问答、待审批行动草案 | 不直连 SAP/数据库；无生产写回 |
| Phase 3：审批式协同 | Huyan → Core/AI 的审批工作流 | 人工批准后的任务提交、状态回传、决策记忆闭环 | 每项写入动作需独立授权、幂等与补偿设计 |
| Phase 4：有限自动化 | 已验证的低风险、可逆场景 | 基于策略的受限自动化和异常升级 | 不可逆/高风险动作仍必须人工批准 |

### 12.2 接口原则

- Huyan 仅通过版本化、认证、审计完备的 API 与 Core/AI 连接；不得直接访问其数据库、文件系统、内部管理接口或基础设施。
- Core 每次请求都重新执行数据权限；Huyan 的 Gateway 权限快照用于上下文传递，不能取代 Core 的权威判断。
- AI 返回内容必须包含模型/Agent 版本、来源、时间、置信度或不确定性，并受同一数据范围约束。
- 所有动作接口以“建议 → 审批 → 提交 → 回执 → 审计”为标准状态机设计；禁止“生成建议即执行”。
- 新增连接器应先在隔离环境验证契约、权限拒绝、错误处理、限流、可观测性和回滚，再申请生产接入。

### 12.3 首批连接器优先级

1. **Gateway Identity → Huyan：** CEO/Founder 受控路由与最小会话上下文。
2. **Core → Dashboard：** 经营健康与 CEO 指标的只读、口径可追溯数据契约。
3. **Core → Knowledge Brain：** 具备 ACL、分类和引用元数据的受控知识来源。
4. **AI/FoxBrain → Agent Hub：** 受版本、权限、限额和审计治理的分析/建议调用。
5. **Approval Service → Core/AI：** 在人工批准后提交受限任务并获取回执；此项不属于第一阶段启用范围。
