## 1. Sprint Goal

Safely connect FoxBrain Enterprise Sync Engine to a real SAP Business One data source through a read-only path.

The objective is:

```text
SAP B1 Production
↓
Read-only Replica / Backup / Secure Export
↓
FoxBrain Sync Engine
↓
Staging
↓
Validation and Reconciliation
↓
Manual Approval
↓
Data Lake
↓
CEO Brain
```

This Sprint is a controlled rollout, not a direct production cutover.

---

## 2. Non-Negotiable Safety Rules

- Do not install FoxBrain, Python, AI, or experimental software on the SAP production server.
- Do not write to the SAP production database.
- Do not use SAP administrator credentials.
- Use a dedicated read-only account with SELECT-only permission.
- Prefer a replicated database, restored backup, or secure export directory.
- Do not commit credentials, IP addresses, ports, certificates, or passwords to GitHub.
- Keep scheduled synchronization disabled until explicit approval.
- The first real sync must be dry-run only.
- Failed validation must block publish.
- A failed FoxBrain sync must never affect SAP operation.

---

## 3. Supported Rollout Modes

Priority order:

1. Independent SAP replica server
2. Restored nightly backup on a separate server
3. Secure export directory generated by SAP-side scheduled task
4. Direct read-only database connection only when the above are unavailable and security approval is explicit

The implementation must record the selected mode.

## 5. Read-Only Security Verification

Codex must provide a verification command or script that checks:

- Connection succeeds.
- Account can execute SELECT.
- INSERT is denied.
- UPDATE is denied.
- DELETE is denied.
- DDL is denied.
- Connection timeout is configured.
- Query timeout is configured.
- Source identity and database name are recorded without exposing secrets.

Store the result as an audit record.

---

## 6. Real Data Dry-Run

First real run must:

- Read data only.
- Write only to FoxBrain staging tables.
- Not publish to Data Lake.
- Not advance the production sync cursor permanently.
- Produce row counts and reconciliation results.

Required initial domains:

```text
sales
inventory
products
brands
stores
employees
suppliers
customers
```

If some domains are not yet mapped, report them as pending rather than fabricating results.

---

## 7. Reconciliation Requirements

For every domain compare:

- source row count
- staged row count
- accepted row count
- rejected row count
- duplicate row count
- null-key count
- amount totals where applicable
- quantity totals where applicable
- earliest and latest source timestamps

Sales reconciliation:

```text
sales rows
sales amount
sales quantity
gross profit when available
```

Inventory reconciliation:

```text
inventory rows
inventory quantity
inventory retail amount
inventory cost amount when available
```

Any unexplained difference must block publish.

## 15. Rollback

Rollback must:

- Remove or supersede the FoxBrain published batch only.
- Never modify SAP source data.
- Restore previous FoxBrain cursor when safe.
- Record the reason and operator.
- Preserve audit history.
