# VAFOX M4.3.1 Product Asset Ingestion MVP Report

**验证日期：** 2026-07-21

**范围：** KAILAS 第一批产品知识资产的设计验证与导入就绪性测试。

**结论：** **CONDITIONAL PASS（可进入受控资料导入和人工审核；尚未发布任何产品知识）。** 当前仓库中没有可审核的 KAILAS 原厂产品资料、带版本的产品目录或 SAP 目录快照。因此，本次不会虚构型号、技术参数、价格、SKU 或产品适配结论，也不会把 M4.2 的品牌级种子内容冒充产品级证据。

| 验收项 | 状态 | 本次可验证结果 | 发布门槛 |
| --- | --- | --- | --- |

| Product Entity 生成 | READY | 已定义字段级提取/审核表和稳定 ID 规则。 | `brand`、`category` 与至少一个主来源齐备；型号未知则保持 `unresolved`。 |

### 2.2 批次接收清单（阻塞发布）

在受控导入前，资料所有者须为每一个具体型号提交原厂 PDF、DOCX、XLSX 或 Markdown（或经批准的不可变归档副本），并附文档版本/发布日期。资料应至少覆盖：产品名/型号、类目、技术说明、规格、适用条件及限制。SAP 目录快照必须由现有只读零售接口在授权范围内提供；不得通过本 MVP 直接连接、写入或修改 SAP。

每份接收文件将计算 `content_sha256`，并生成 `document_id`、原文件名、MIME、导入人、导入时间、解析器版本和 ACL。PDF 使用 `page=<n>&chars=<start>-<end>`；DOCX 使用 `heading=<heading>&paragraph=<n>`；XLSX 使用 `sheet=<sheet>&range=<range>`；Markdown 使用 `heading=<heading>&lines=<start>-<end>`。文档版本缺失、页/段定位缺失、哈希缺失、加密不可审查或无法提取文本时，资产必须隔离，不能进入 `approved`。

### 字段级生成规则

| 字段 | 生成规则 | 不可接受的替代 |
| --- | --- | --- |
| `brand`、`series`、`model`、`category` | 从原厂产品标题、型号字段和目录分类提取；`series/model` 不明确时填 `unresolved`。 | 从 SKU 显示名称、营销简称或模型猜测。 |
| `technology`、`specification` | 原文摘取并以字段级 Citation 连接；单位与适用变体原样保留。 | 将“专业”“高性能”等营销语改写成技术机制或性能参数。 |
| `scenario`、`customer_profile` | 使用下列受控标签；仅在资料明确支持时标为 `suitable`，否则为 `conditional` 或待审。 | 从品牌定位、客户住址或类目名称推断。 |
| `selling_points`、`recommendation` | 分别记录事实、适用条件、限制和 `evidence_refs`；推荐永远需要员工确认。 | 保证安全、适配、库存、价格或成交结果。 |

## 4. 只读 SKU Mapping

映射方向固定为 **SAP SKU → Product Entity**，不会反向创建/更改 SAP 商品。导入资料中发现的 SKU 只是候选键；映射按 `sap_sku + organization + channel + as_of` 查询授权目录快照，再核对型号、颜色/尺码等变体和有效期。

| 字段 | 约束 |
| --- | --- |
| `sap_sku` | SAP 权威标识；本报告未写入示例 SKU，以免制造不存在的主数据。 |
| `product_id` | 只引用第 3 节已审核的稳定实体；一件产品可有多个变体 SKU。 |
| `source_snapshot_id`、`as_of` | 必填，指向只读目录快照及其时点。 |
| `mapping_status` | 仅 `verified` 可成为零售核验或商品推荐输入；`unresolved`、`ambiguous`、`deprecated` 均不得称为可售。 |
| `effective_from/to` | 映射有效期，禁止静默覆盖历史映射。 |

## 5. Citation 完整性与场景标签

### 5.1 Citation 最小记录

每个知识点使用 `citation_id` 关联以下不可缺少的信息：`document_id`、`source_file`、`source_version`、`content_sha256`、`page`、`paragraph_location`、`locator`、`claim_path`、`chunk_id`、`captured_at` 与 `extractor_version`。非分页格式的 `page` 明确记为 `not_applicable`，但段落/行/单元格位置仍为必填；不能用空字符串代替。

```json
{
  "citation_id": "cit:kailas:<document-id>:<claim>",
  "source_file": "<original-filename>",
  "source_version": "<manufacturer-version-or-publication-date>",
  "content_sha256": "<sha256>",
  "page": "<page-number|not_applicable>",
  "paragraph_location": "<heading-and-paragraph|sheet-and-range|line-range>",
  "claim_path": "technology[0].benefit"
}
```

### 5.2 受控 Scenario Tag

| 维度 | 可用标签 | 审核规则 |
| --- | --- | --- |
| 活动 | `hiking`（徒步）、`mountaineering`（登山）、`high_altitude`（高海拔）、`camping`（露营） | 必须是资料支持的活动；高海拔不等于专业能力。 |
| 用户 | `newcomer`（新人）、`intermediate`（进阶）、`professional`（专业） | 这是能力适配而非客户身份；资料不足时不标注。 |
| 环境 | `shenzhen`（深圳）、`western_sichuan`（川西）、`tibet`（西藏）、`snow_mountain`（雪山） | 只能表示产品资料所支持的条件；不得由住址推断。 |

`fit=conditional` 或 `fit=not_suitable` 必须携带限制及 Citation。任何高海拔、雪山或多日行程的输出均须提示人工确认海拔、天气、路线、时长、健康/经验、尺码和实际可售状态。

## 7. Recommendation Test：第一次川西、预算 ¥8,000

### 输入与预期安全响应

```json
{
  "activity": "hiking",
  "environment": ["western_sichuan"],
  "customer_level": "newcomer",
  "budget": {"currency": "CNY", "max": 8000},
  "request_type": "advisory_only"
}
```

| 输出项目 | 结果 |
| --- | --- |
| 产品组合 | 返回 `pkg:kailas:western-sichuan-7d` 的角色级候选：防风雨外层、保暖层、抓绒或软壳层、徒步鞋、背包；未返回 SKU、价格、库存或“有货”承诺。 |
| 推荐理由 | 显式需求是第一次川西、多日徒步和 ¥8,000 上限；组合用于提示分层、防护、步行支撑和负载携带需求。它不是任何 KAILAS 具体型号的性能或安全结论。 |
| 引用来源 | `src:m43-schema` 与 `src:m431-pipeline`（组合/安全/只读契约）；每个未来产品理由必须另外附原厂产品 Citation。 |
| 置信度 | `low`：缺少季节、路线、海拔、天气、行程、尺码、体感、健康/经验详情，且没有审批产品资料、验证 SKU 或实时价格/可售快照。 |
| 必须人工确认 | 路线与海拔、出行日期/天气、健康与高反风险、尺码/试穿/试背、睡眠与补水需求、源文件版本、SKU 映射、门店渠道、实时价格和库存。 |

该测试证明缺证据时系统会降级为“员工核实”，而不是用品牌资料、预算或场景自动生成销售结论。即使以后目录价格合计在预算内，仍需依赖同一时点、同一授权范围的只读 Retail Brain 事实；本 MVP 不读取、更不写入该事实。

## 8. 测试与发布判定

设计和现有品牌知识回归验证通过；资产发布验证**故意阻断**，因为没有可审查的产品主来源和 SAP 快照。建议下一步由资料所有者将正式文档放入受控导入队列，运行解析/哈希/locator 检查，人工补全字段并审核，再分别执行 SKU 和推荐回归。

| 测试 | 预期 | 结果 |
| --- | --- | --- |
| M4.2 品牌知识与 M4.1 知识基础回归 | KAILAS 品牌级检索、Citation、ACL 不回归 | PASS |
| Python 编译检查 | 服务与测试源可编译 | PASS |
| Git 空白检查 | 本次文档不含空白错误 | PASS |
| KAILAS 实体发布门 | 每项实体有原厂主来源、hash、版本、locator 与审核 | BLOCKED（正确的安全行为） |
| SAP SKU 映射门 | 授权只读目录快照、唯一且有效的 `verified` 映射 | BLOCKED（未提供快照；未访问 SAP） |
| 川西 SKU 推荐门 | 已批准实体 + `verified` SKU + 实时价格/可售 | BLOCKED（返回角色级人工核验建议） |

## 9. 最终边界确认

- 未修改 SAP，未修改 Core 生产同步，未创建 SAP 写入路径。
- 未部署服务、未迁移数据、未将草稿资产写入可检索索引。
- 未自动下单、预留库存、报价、自动销售或触达客户。
- 未把无来源的产品技术、规格、型号、价格、库存或安全适配当作事实。

**发布建议：** 收到版本化 KAILAS 原厂产品资料及授权 SAP 只读目录快照后，按本报告第 2–5 节完成字段级 Citation、人工审核和 `verified` 映射；在此之前，保持本 MVP 为设计/测试状态。
