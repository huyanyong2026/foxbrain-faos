# VAFOX 2.0 企业 AI 操作系统总体架构蓝图

> **版本**：VAFOX 2.0  
> **定位**：未来十年的数字战略与企业智能化架构基线  
> **设计原则**：品牌一致、移动优先、数据可信、权限最小化、AI 可治理、能力可演进。

## 一、VAFOX 2.0 总体定位

VAFOX 2.0 不再仅被定义为一家户外零售企业，而是升级为面向人、商品、数据、知识与组织协同的 **Outdoor Intelligence Ecosystem（户外智能生态系统）**。

其核心载体是 **VAFOX OS**：一个以企业数据与知识为基础、由 AI Agent 协作执行、由 Control Plane 统一治理、由 CEO 决策闭环持续驱动的企业 AI 操作系统。

VAFOX OS 将实体零售、户外商品、供应链与客户经营能力，与企业经营数据、组织知识、人才发展和 AI 劳动力连接为统一智能网络，实现“感知—理解—决策—执行—复盘—学习”的持续循环。

### 战略目标

1. **经营智能化**：让关键经营事实及时进入 CEO 决策视野，并转化为可执行行动。
2. **组织智能化**：让每一位员工获得受控、可信、可学习的 AI 工作助手。
3. **知识资产化**：让文件、合同、制度、经验、复盘与决策形成可检索、可引用、可沉淀的企业记忆。
4. **数据产品化**：让 SAP B1 等业务系统中的数据成为可治理、可复用、可审计的数据资产。
5. **AI 劳动力化**：让 Agent 成为有明确职责、权限边界、输入输出与审计记录的数字岗位。

## 五、四大 Brain 体系

### 1. CEO Brain

CEO Brain 是 VAFOX 的经营决策中枢。它将核心经营数据、重要报告、关键知识、风险信号与待决事项组织为 CEO 可理解、可追溯、可行动的上下文。

- **关注对象**：营收、利润、库存、现金、供应链、客户、组织、风险与战略事项。
- **输入**：Core Data Brain 指标、Report Agent 结论、关键知识、CEO 指令与授权范围内的组织信息。
- **输出**：经营简报、异常提醒、决策选项、行动任务、决策记录和复盘素材。
- **原则**：AI 提供证据与建议，最终经营判断、审批和高风险行动由具备权限的人完成。

### 2. Knowledge Brain

Knowledge Brain 是企业长期记忆与可验证知识底座，负责将非结构化信息转化为可检索、可引用、可授权使用的知识资产。

- **知识来源**：文件、合同、制度、产品资料、项目材料、会议纪要、经验、复盘及决策记录。
- **关键能力**：分类、版本管理、权限继承、语义检索、来源引用、生命周期管理与知识质量评估。
- **边界**：知识回答必须带有来源和权限校验；无来源、过期或无权限内容不得作为确定性结论输出。

### 3. People Brain

People Brain 是组织与人才能力模型，连接员工角色、技能、目标、学习、绩效反馈、岗位知识和成长路径。

- **关注对象**：人、岗位、团队、技能、成长目标、学习任务与组织能力缺口。
- **关键能力**：能力画像、岗位知识包、个性化学习建议、组织能力分析与人才发展闭环。
- **隐私原则**：个人敏感信息采用严格分类与最小可见范围；不得将未经授权的人事数据用于无关的 AI 推断。

### 4. Core Data Brain

Core Data Brain 是企业经营事实的可信底座，负责把 SAP B1 和其他合法业务来源转化为统一、可解释、可审计的数据服务。

- **核心对象**：商品、客户、订单、库存、采购、财务、门店、渠道、供应商和组织主数据。
- **关键能力**：数据接入、标准模型、指标口径、主数据治理、质量监控、血缘追踪、数据 API 与权限控制。
- **原则**：业务事实以权威源系统为准；分析、缓存和 AI 结果不得反向篡改 SAP B1 原始记录。

## 六、Agent 体系

Agent 是 VAFOX AI Workforce 中具备数字岗位定义的智能执行单元。每个 Agent 必须被配置职责、允许输入、受控输出、工具范围、数据范围、审批条件与审计策略。

| Agent | 职责 | 输入 | 输出 | 权限 |
|---|---|---|---|---|
| **Health Agent** | 持续监测平台、数据任务、集成与 Agent 运行健康。 | 服务指标、任务状态、日志、告警规则。 | 健康评分、异常告警、初步诊断、升级工单。 | 只读观测；可创建告警与工单；不得修改生产数据。 |
| **Connectivity Agent** | 管理与观察系统连接、接口可用性、同步状态及失败重试建议。 | 集成配置元数据、接口状态、消息队列事件。 | 连接状态、失败报告、重试建议、依赖影响分析。 | 受控执行重试；连接凭据不可见；高风险变更须人工审批。 |
| **Report Agent** | 将可信数据转换为标准经营报告和面向角色的摘要。 | 已认证指标、报告模板、时间范围、授权上下文。 | 日报、周报、月报、异常说明与数据引用。 | 只读访问经授权的数据产品；不得自行更改指标口径。 |
| **Data Agent** | 执行数据查询、质量检查、血缘解释与受控的数据准备任务。 | 数据请求、语义模型、质量规则、权限令牌。 | 查询结果、质量问题、指标解释、可审计数据集。 | 按行列与目的授权读取；写入仅限隔离的派生层与审批工作流。 |
| **CEO Agent** | 为 CEO Brain 汇总信号、形成决策准备材料并跟踪授权行动。 | 报告、风险、战略知识、CEO 指令、授权数据。 | CEO 摘要、行动选项、任务建议、决策复盘上下文。 | 仅 CEO/明确委托范围；不可代表 CEO 作最终审批或对外承诺。 |
| **People Agent（未来）** | 支持员工成长、岗位辅导和组织能力发展。 | 角色、技能、学习资源、员工授权反馈。 | 个性化成长计划、学习建议、能力洞察。 | 仅处理授权的人才信息；敏感人事操作须人工审批。 |

### Agent 运行原则

1. **先授权、后执行**：无身份、无策略、无可审计上下文的任务不得执行。
2. **最小权限**：Agent 仅获得完成当前任务所需的最小工具与数据范围。
3. **人类在环**：资金、主数据、合同、对外沟通、人事与高风险生产变更必须设置审批关卡。
4. **结果可验证**：报告、分析和知识回答应关联数据血缘或知识来源。
5. **全程可审计**：记录请求者、Agent、模型、工具、数据范围、输出、审批和结果状态。

## 七、数据流架构

```text
SAP B1
  ↓  权威业务数据接入、校验与标准化
Core Data Brain
  ↓  受控数据产品、指标与语义上下文
Data Agent
  ↓  数据分析、质量解释与结构化结果
Report Agent
  ↓  标准报告、趋势、异常与证据引用
CEO Agent
  ↓  决策摘要、行动选项与任务跟踪
Huyan
```

### 数据流治理要求

- SAP B1 是核心业务记录的权威系统；数据通过受控集成进入 Core Data Brain。
- Core Data Brain 负责口径统一、质量校验、血缘记录与数据产品发布。
- Data Agent 仅在已授权的数据范围内查询和处理，不直接写回源系统。
- Report Agent 以已认证指标和模板生成报告，明确数据时间范围、来源与异常。
- CEO Agent 将报告转化为决策准备材料；Huyan 呈现信息、承接指令并保存决策上下文。
- 任何写回业务系统的行动均应走独立审批、验证与审计流程，不能由报告链路自动完成。

## 八、知识流架构

```text
文件 / 合同 / 制度 / 经验 / 复盘 / 决策
                    ↓
             Knowledge Brain
                    ↓
          AI Agent（检索、理解、辅助）
                    ↓
                企业成长
```

知识流不是简单的文档上传流程，而是企业经验的治理循环：

1. **采集与归档**：接入有效来源，保留所有者、版本、密级、有效期和来源信息。
2. **理解与组织**：通过分类、标签、关联和语义索引构建可用知识上下文。
3. **受控使用**：AI Agent 在访问前校验身份、知识权限、用途与时效性。
4. **引用与反馈**：输出关联来源，员工可纠错、补充、评价或提交新经验。
5. **沉淀与迭代**：高价值复盘与决策被固化为组织知识，持续提升企业学习速度。

## 九、人才成长架构

```text
员工
  ↓
People Brain
  ↓
AI 成长助手
  ↓
能力提升
  ↓
组织能力增强
```

### 人才成长闭环

- **员工**：明确角色、目标、当前能力与可选择的发展路径。
- **People Brain**：形成岗位能力模型、学习资源图谱和经授权的成长数据。
- **AI 成长助手**：根据岗位与目标推荐学习、提供情境辅导、生成实践任务并收集反馈。
- **能力提升**：以学习完成、实践产出、主管反馈和自我反思共同验证，不以单一 AI 评分替代人类判断。
- **组织能力增强**：聚合匿名化、最小必要的能力趋势，识别关键岗位缺口并改善培训与知识供给。

## 十、身份权限体系

VAFOX OS 以 **gateway.vafox.com** 为统一身份与访问控制入口，确保用户、服务、应用与 Agent 在同一安全模型下运行。

| 能力 | 定义 | 架构要求 |
|---|---|---|
| **Gateway** | 统一身份、会话、令牌与访问入口。 | 所有跨域访问、Agent 工具调用和敏感服务访问均通过 Gateway。 |
| **RBAC** | 基于角色的访问控制。 | 以 CEO、管理者、员工、数据管理员、AI 运营、安全审计等角色提供基础授权。 |
| **ABAC** | 基于属性的访问控制。 | 叠加组织、地域、数据密级、资源归属、设备状态、时间、用途与风险等级。 |
| **MFA** | 多因素认证。 | 对管理、敏感数据、高风险操作与异常会话强制启用。 |
| **Audit** | 审计与可追溯。 | 记录认证、授权决策、数据访问、Agent 调用、审批、配置变更与异常事件。 |

权限判定应同时考虑“谁在访问、访问什么、为何访问、在何种条件下访问”。任何权限提升、跨域共享、批量导出与高风险操作必须具备明确审批与可撤销机制。

## 十一、安全边界

### 1. SAP 隔离

- SAP B1 保持为核心交易与业务记录系统，不直接暴露给通用 AI 对话或公网入口。
- 通过受控连接器、服务账户、只读副本或审批 API 进行集成，限制写回能力。
- 对财务、库存、采购等高敏感操作设置强制人工复核和双重审计。

### 2. Core 隔离

- Core Data Brain 与外部体验层、非可信工作负载分区部署。
- 原始数据、派生数据、脱敏数据和公开数据按分类分层存储与访问。
- 仅通过认证的数据产品、查询服务和策略网关向上层提供数据。

### 3. Agent 权限

- 每个 Agent 使用独立身份、独立工具清单和独立数据范围。
- 禁止共享高权限凭据；禁止 Agent 自行扩大权限或绕过 Control Plane。
- 对写操作、外发操作、支付、人事、合同和生产变更实施审批与回滚机制。

### 4. 数据权限

- 数据按业务敏感度分级，执行行级、列级、字段级与目的限制。
- 对个人信息、财务数据、合同内容和商业机密采取脱敏、加密、留存期限与导出控制。
- 所有数据消费均保留血缘、访问记录和授权依据。

### 5. AI 可信

- AI 输出必须区分事实、推断、建议与未知；关键结论应能回溯至数据或知识来源。
- 对提示注入、越权工具调用、敏感信息泄露、幻觉、高风险内容和异常行为进行防护与监测。
- 模型、提示词、工具、知识版本与评估结果纳入 Control Plane 的版本治理和发布流程。

## 十二、移动优先战略

VAFOX 2.0 采取 **手机为主、网页辅助** 的体验战略。

- **手机为主**：面向一线员工、门店、管理者和 CEO 的高频任务，以通知、查看、审批、问答、提醒、行动跟踪和快速反馈为核心。
- **网页辅助**：面向复杂配置、深度分析、批量操作、数据治理、Agent 运营、审计与管理工作台。
- **一致体验**：同一身份、相同权限语义、连续任务上下文和统一品牌设计在手机与网页之间保持一致。
- **离线与弱网考虑**：关键现场流程应设计为可恢复、可同步、可追溯，避免因网络波动丢失业务上下文。

## 十三、未来十年路线图

| 阶段 | 战略主题 | 关键建设 | 阶段成果 |
|---|---|---|---|
| **Phase 1** | 基础安全 | Gateway、统一身份、MFA、RBAC/ABAC、审计、SAP 与 Core 隔离、数据分级。 | 建立可信、可控的企业 AI 使用底线。 |

| **Phase 3** | Agent Organization | Health、Connectivity、Data、Report 等 Agent 上线，形成工作流与人机协作机制。 | AI Workforce 开始承担明确、可审计的数字岗位。 |
